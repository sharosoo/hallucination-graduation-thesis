\section{실험 설정}

본 절에서는 환각 탐지 실험 절차를 설명한다. 실험의 핵심은 모델이 새 답변을 생성한 뒤 그 정오를 사후에 판정하는 방식이 아니라, 데이터셋이 이미 정답 답과 환각 답을 명시적으로 구분해 제공한다는 점을 활용해 두 후보를 같은 질문 아래에서 paired로 비교하는 데 있다. 이렇게 하면 (i) 새 답이 정답인지를 판정하기 위한 문자열 일치 heuristic, (ii) 별도 LLM에게 정오를 묻는 LLM-as-judge 방식 모두에 의존하지 않고, 이미 annotation으로 구분된 후보 쌍을 대상으로 uncertainty와 corpus-grounded feature가 환각 탐지에 어떤 신호를 주는지 분석할 수 있다.

\subsection{데이터셋 구성}

실험에는 TruthfulQA와 HaluEval-QA 두 데이터셋을 사용하였다. 두 데이터셋 모두 질문마다 정답 후보와 잘못된(혹은 환각) 후보를 명시적으로 제공하므로 paired 평가에 적합하다. TruthfulQA에서는 각 질문의 \texttt{correct\_answers}와 \texttt{incorrect\_answers}에서 하나의 정답 후보와 하나의 오답 후보를 결정론적으로 선택하였다. HaluEval-QA에서는 데이터셋이 제공하는 \texttt{right\_answer}와 \texttt{hallucinated\_answer}를 그대로 사용하였다.

각 질문은 하나의 prompt 단위로 다루며, 한 prompt에서 정확히 두 개의 candidate row를 만든다. 하나는 정답 후보, 다른 하나는 환각 후보이다. 최종 실험 데이터는 prompt 5,815개, candidate row 11,630개로 구성된다. HaluEval-QA가 prompt 5,000개와 row 10,000개, TruthfulQA가 정제 후 prompt 815개와 row 1,630개를 제공한다.

\subsection{정답성 라벨링}

정답성 라벨은 모델 출력이나 후처리 judge에서 오지 않는다. 각 candidate row의 주 라벨은 데이터셋 annotation이 제공하는 \texttt{is\_hallucination} 이진 값이며, 이 값이 최종 환각 탐지 평가의 기준이다. 즉 본 연구는 데이터셋 annotation을 ground truth로 받아들이고, 모델 출력 자체에서 라벨을 유도하지 않는다. 이때 Semantic Entropy는 prompt 단위 값이므로 같은 질문의 정답 후보와 환각 후보가 같은 SE 값을 공유한다.

\subsection{모델 scoring 절차}

모델은 Qwen2.5-3B 계열 causal LM을 사용한다. 같은 계열 tokenizer(vocab size 151{,}936)의 full-vocabulary logits을 그대로 보존하기 위해 생성 산출물은 \texttt{float16} sidecar parquet에 저장한다. 실험은 두 종류의 모델 출력을 사용한다.

\begin{itemize}
    \item \textbf{Free sampling}. Semantic Entropy 계산을 위해 각 prompt에 대해 answer-only protocol로 N=10개의 짧은 답변을 sampling한다. 이는 prompt-level 신호의 입력이다.
    \item \textbf{Teacher-forced scoring}. candidate-level feature 계산을 위해 각 정답/환각 후보를 teacher-forced 방식으로 scoring한다. 모델이 후보 답을 새로 생성하지는 않으며, 주어진 후보 답의 각 token 위치에서 full-vocabulary logits, token별 \texttt{logsumexp}(분배 함수 $-\log Z$), 선택된 token의 logit과 log probability를 모두 기록한다.
\end{itemize}

이 구분은 이후 결과 해석에서 중요하다. Semantic Entropy는 같은 prompt의 두 candidate row에 같은 값이 복사되지만, candidate-level diagnostic(NLL, logit variance, confidence margin)은 후보마다 값이 다르고, Semantic Energy는 cluster 단위로 정의된다.

Answer-only protocol은 \texttt{generation.max\_new\_tokens=96}, \texttt{answer\_only.max\_answer\_tokens=64}, \texttt{answer\_only.max\_invalid\_attempts=32}, \texttt{fail\_on\_max\_new\_tokens=true}로 고정한다. 이는 HaluEval-QA의 긴 context tail prompt가 더 짧은 token 한도(32/64/8)에서 bounded resampling을 모두 소진해 stage가 중단되는 현상을 해결하기 위한 값이며, 짧은 답변 절차의 의도는 유지하되 multi-entity 답변을 안전하게 표현할 여유를 준다.

\subsection{특징 계산}

본 연구는 세 종류의 특징군을 사용한다.

\paragraph{Semantic Entropy.}
Semantic Entropy는 한 prompt에서 얻은 N=10 답변을 \texttt{microsoft/deberta-large-mnli} 기반 bidirectional NLI entailment로 semantic cluster에 묶은 뒤, 표본 sequence log-likelihood를 cluster 단위로 log-sum-exp로 집계해 cluster probability mass를 만들고 그 위에서 entropy를 계산한다. 산출물에는 \texttt{semantic\_entropy\_nli\_likelihood}, \texttt{semantic\_entropy\_cluster\_count}, NLI model reference, 표본 / cluster log-likelihood가 함께 기록된다. SE는 prompt-level feature이고, paired 설정에서 정답 후보와 환각 후보가 같은 SE 값을 받는다.

\paragraph{Semantic Energy와 candidate-level diagnostic.}
Semantic Energy는 Ma 등(2025) 식 (11)--(14)를 그대로 따른다. 동일한 N=10 답변과 SE 단계의 NLI semantic cluster를 재사용하며, cluster를 다시 계산하지 않고 \texttt{(prompt\_id, sample\_index)}로 join한다. 식은 다음과 같다.
\begin{align}
\tilde{E}(x_t^{(i)}) &= -z_\theta(x_t^{(i)}) \quad \text{(식 13: token energy = $-$ selected-token logit)} \\
E(x^{(i)}) &= \frac{1}{T_i}\sum_{t=1}^{T_i} \tilde{E}(x_t^{(i)}) \quad \text{(식 11: sample energy)} \\
E_{\text{Bolt}}(\mathbb{C}_k) &= \sum_{x^{(i)} \in \mathbb{C}_k} E(x^{(i)}) \quad \text{(식 12: cluster total energy)} \\
U(\mathbf{x}) &= \sum_{k=1}^{K} p(\mathbb{C}_k)\, E_{\text{Bolt}}(\mathbb{C}_k) \quad \text{(식 14: 최종 uncertainty)}
\end{align}
여기서 $p(\mathbb{C}_k) = \sum_{x^{(i)} \in \mathbb{C}_k} \bar{p}(x^{(i)} \mid q)$는 식 (8)의 likelihood-based cluster probability로, SE 단계에서 산출된 값을 그대로 상속한다. 이는 Ma 등(2025) §2.2가 인용하는 Farquhar 등(2024)의 cluster-likelihood weighting과 일치한다. 부호 규약은 lower energy = higher reliability(높은 energy = 높은 uncertainty)이다.

한편 teacher-forced candidate scoring에서 얻은 token-level $-\log Z$의 평균, NLL의 평균, logit variance, confidence margin은 candidate-level Boltzmann-style energy diagnostic으로 별도 보존하며, Semantic Energy와 같은 열에 합치지 않는다.

\paragraph{Corpus-grounded feature.}
Corpus feature는 후보 답변 텍스트에서 entity를 추출한 뒤 Infini-gram-compatible count backend로 entity frequency와 entity-pair co-occurrence를 query하여, raw count, log-transformed continuous score, 그리고 3-bin / 5-bin / 10-bin corpus support label(\texttt{corpus\_axis\_bin}, \texttt{\_5}, \texttt{\_10})을 함께 기록한다. 본 실험의 기본 backend는 \texttt{InfinigramLocalEngineBackend} 위의 \texttt{v4\_dolmasample\_olmo} index(16B Dolma sample tokens, OLMo tokenizer)이며, \texttt{s3://infini-gram-lite/index/v4\_dolmasample\_olmo/}에서 미리 로컬 디스크로 동기화한 뒤 \texttt{InfiniGramEngine.count\_cnf} AND query로 pair count를 얻는다. 공개 REST API(\texttt{https://api.infini-gram.io/})는 같은 cache schema의 fallback으로 둔다. 이 feature는 외부 corpus에서 후보 답변의 지지 가능성을 간접적으로 나타내는 proxy이며, 모델의 hidden-state나 정답성 라벨을 사용하지 않는다. 낮은 frequency와 zero co-occurrence는 환각 라벨이 아니라 지표 신뢰도를 조건화하는 binning 변수이다.

\subsection{실험 단계 (S0--S9)}

전체 실험은 \texttt{run\_pipeline.py}가 자동 실행하는 9단계(S0--S9)로 구성된다. 각 단계는 입출력 파일이 명시되어 있어 단일 단계 재실행, 산출물 검증, 동일 입력에서의 결정론적 재현이 가능하다.

\begin{enumerate}
    \item[\textbf{S0--S1.}] \textbf{Paired 데이터셋 구성}. TruthfulQA와 HaluEval-QA에서 prompt별 정답 후보와 환각 후보를 구성한다.
    \item[\textbf{S2--S3.}] \textbf{모델 출력 수집}. 각 prompt에 대해 N=10 답변을 free sampling하고, 각 후보 답변을 teacher-forced full-logit으로 scoring한다. 정답성 라벨은 데이터셋 annotation만 사용해 확정한다.
    \item[\textbf{S4.}] \textbf{Semantic Entropy 계산}. prompt-level N=10 표본을 NLI semantic cluster로 묶고 likelihood-based SE와 cluster metadata를 계산한다.
    \item[\textbf{S5.}] \textbf{Corpus support 축 계산}. 후보 답변에서 entity frequency와 entity-pair co-occurrence를 continuous score와 bin label(3-bin / 5-bin / 10-bin)로 계산한다.
    \item[\textbf{S6.}] \textbf{Semantic Energy 계산}. 생성된 답변의 semantic cluster와 selected-token logit에서 Semantic Energy를 계산하고, candidate-level logit diagnostic은 별도로 계산한다.
    \item[\textbf{S7.}] \textbf{Feature table 결합}. prompt-level SE를 두 candidate row에 복사하고, candidate-level Energy / corpus feature를 candidate id 기준으로 결합한다.
    \item[\textbf{S8.}] \textbf{Fusion 및 baseline 평가}. SE-only, Energy-only, logit-diagnostic-only, corpus-only, global learned fusion(no corpus / with corpus), corpus-bin weighted fusion, axis-interaction fusion, sklearn 비선형 결합기를 비교한다.
    \item[\textbf{S9.}] \textbf{Robustness 분석}. prompt-grouped bootstrap, leave-one-dataset-out, corpus-bin reliability, binning sensitivity(3/5/10-bin), threshold / calibration 진단을 수행한다.
\end{enumerate}

\subsection{평가 지표와 해석 원칙}

공통 baseline 표에는 AUROC, AUPRC, F1 등 표준 분류 지표를 보고한다. 학습 라벨은 데이터셋 annotation의 \texttt{is\_hallucination} 이진 값이다. 주 분석 축은 corpus support bin(3-bin 기본, 5-bin / 10-bin sensitivity)이며, 각 bin 안에서 SE / Energy / logit-diagnostic의 AUROC와 paired win-rate를 보고한다.

다만 이 candidate-row 종합 지표는 모든 feature에서 같은 의미를 갖지 않는다. Semantic Entropy는 prompt-level feature이므로, 같은 prompt의 정답 후보와 환각 후보가 같은 SE 값을 가진다. SE-only의 candidate-row AUROC가 0.5에 묶이는 것은 SE가 무의미하다는 뜻이 아니라, prompt-level feature를 candidate-row 순위 평가에 적용한 구조적 결과이다.

Semantic Energy와 logit diagnostic은 candidate-level 또는 cluster-level 신호이지만, 그 신뢰도가 corpus support 조건에 따라 달라질 수 있다. 따라서 핵심 평가는 각 corpus support bin에서 환각 후보가 짝지어진 정답 후보보다 더 높은 위험 점수를 받는지 확인하는 matched-pair metric이다. 구체적으로 (i) hallucinated $-$ correct paired delta, (ii) paired win-rate, (iii) prompt-grouped bootstrap 95\% 신뢰구간, (iv) leave-one-dataset-out 일반화, (v) threshold / calibration sensitivity, (vi) 3/5/10-bin binning sensitivity를 모두 보고한다. Train/test 분할은 prompt 단위로 묶어 같은 prompt의 두 candidate row가 fold를 가로지르지 않게 강제한다.

\subsection{재현 가능성}

본 실험은 \texttt{run\_pipeline.py}가 자동 실행하는 9단계(S0--S9)로 진행되며, 각 단계마다 실행 명령, 기대되는 입출력 파일, validator 명령, git commit hash, Python 버전, platform, UTC timestamp가 manifest와 실행 로그에 기록된다. 산출물 schema는 명시적으로 버전 태그(예: \texttt{semantic\_entropy\_nli\_likelihood\_v1}, \texttt{semantic\_energy\_cluster\_v1}, \texttt{corpus\_axis\_counts\_v1})를 가지며, schema가 다른 기존 산출물은 ``재실행 시 건너뛰기'' 대상에서 제외된다. 모든 최종 산출물은 데이터셋 annotation 라벨을 그대로 사용하며, heuristic 문자열 일치나 LLM-as-judge 기반 정오 판정은 본 연구의 라벨 출처로 인정하지 않는다. 최종 feature table은 11{,}630개 candidate row에 대해 SE, Semantic Energy, candidate-level diagnostic, corpus support bin, backend provenance를 모두 포함하며, train fold만으로 fit된 corpus-conditioned fusion이 held-out fold에서 평가된다.
