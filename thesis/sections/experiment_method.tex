\section{실험 설정}

본 절에서는 본 연구에서 사용한 환각 탐지 실험 절차를 설명한다. 실험의 핵심은 모델이 새 답변을 생성한 뒤 사후에 정답성을 판정하는 방식이 아니라, 데이터셋이 제공하는 정답 후보와 환각 후보를 같은 질문 아래에서 비교하는 것이다. 이를 통해 문자열 휴리스틱이나 LLM-as-judge에 의존하지 않고, 이미 annotation으로 구분된 후보 쌍을 대상으로 불확실성 및 corpus-grounded 특징이 환각 탐지에 어떤 신호를 제공하는지 분석한다.

\subsection{데이터셋 구성}

실험에는 TruthfulQA와 HaluEval-QA 두 데이터셋을 사용하였다. 두 데이터셋은 질문에 대해 정답 후보와 잘못된 또는 환각 후보를 명시적으로 제공하므로, 본 연구의 paired discriminative protocol에 적합하다. TruthfulQA에서는 각 질문의 \texttt{correct\_answers}와 \texttt{incorrect\_answers}에서 하나의 정답 후보와 하나의 오답 후보를 결정론적으로 선택하였다. HaluEval-QA에서는 데이터셋이 제공하는 \texttt{right\_answer}와 \texttt{hallucinated\_answer}를 그대로 사용하였다.

각 질문은 하나의 prompt 단위로 취급되며, 각 prompt에서 정확히 두 개의 평가 row를 만든다. 하나는 정답 후보이고, 다른 하나는 환각 후보이다. 최종 실험 데이터는 5,815개 prompt와 11,630개 candidate row로 구성된다. HaluEval-QA는 5,000개 prompt와 10,000개 row를 제공하고, TruthfulQA는 정제 후 815개 prompt와 1,630개 row를 제공한다.

\subsection{정답성 라벨링}

정답성 라벨은 모델 출력이나 후처리 judge에서 오지 않는다. 각 candidate row의 주 라벨은 데이터셋 annotation이 제공하는 \texttt{is\_correct}/\texttt{is\_hallucination}이며, 이것이 최종 hallucination detection 평가의 기준이다. 아래의 \texttt{NORMAL}, \texttt{HIGH\_DIVERSITY}, \texttt{LOW\_DIVERSITY}, \texttt{AMBIGUOUS\_INCORRECT} 표기는 이전 repo-local baseline을 해석하기 위해 보존한 archived diagnostic 운영 라벨이다. 즉, thesis-valid corpus-axis 분석에서는 이 운영 라벨이 새로운 class ontology가 아니며, annotation-backed correctness와 corpus-axis bin이 주 분석 단위가 된다. 이때 Semantic Entropy는 prompt-level 값이므로 같은 질문의 정답 후보와 환각 후보가 동일한 SE 값을 공유한다.

\subsection{모델 scoring 절차}

모델은 Qwen2.5-3B 계열 causal language model을 사용하였고, 같은 계열 tokenizer (vocab 151{,}936) 의 full-vocabulary logits을 그대로 보존하기 위해 generation artifact는 \texttt{float16} sidecar parquet에 저장한다. 실험은 두 종류의 모델 출력을 사용한다. 첫째, Semantic Entropy 계산을 위해 각 prompt에 대해 answer-only protocol로 N=10개의 짧은 답변을 free sampling한다. 둘째, candidate-level 특징을 계산하기 위해 각 정답/환각 후보를 teacher-forced 방식으로 scoring한다. 즉, 모델이 후보 답을 새로 생성하는 것이 아니라, 주어진 후보 답의 각 토큰 위치에서 full-vocabulary logit, per-step \texttt{logsumexp}, 선택 token logit과 logprobs를 모두 기록한다.

Free sampling은 Semantic Entropy를 계산하기 위한 prompt-level 절차이다. 반면 teacher-forced candidate scoring은 paper-faithful Semantic Energy와 candidate-level diagnostic (negative log probability, logit variance, confidence margin) 의 입력이다. 이 구분은 이후 평가 해석에서 중요하다. Semantic Entropy는 같은 prompt의 두 candidate row에 broadcast되지만, Semantic Energy는 cluster 단위로 share되거나 (paper-faithful 정의 시) candidate 단위로 분리되며, candidate-level diagnostic은 candidate마다 서로 다른 값을 갖는다.

Answer-only protocol은 \texttt{generation.max\_new\_tokens=96}, \texttt{answer\_only.max\_answer\_tokens=64}, \texttt{answer\_only.max\_invalid\_attempts=32}, \texttt{fail\_on\_max\_new\_tokens=true}로 고정한다. 이 캡은 HaluEval-QA의 long-context tail prompt가 더 짧은 캡 (32/64/8) 에서 bounded resampling을 모두 소진해 stage가 abort되는 현상을 해결하기 위해 완화한 값이며, 짧은 답변 절차의 의도는 유지하되 다중 entity 답변을 안전하게 표현할 여유를 둔다.

\subsection{특징 계산}

본 연구는 세 종류의 특징군을 사용한다.

\paragraph{Semantic Entropy.}
Semantic Entropy는 한 prompt에서 얻은 N=10 answer-only free-sample 답변을 \texttt{microsoft/deberta-large-mnli} 기반 bidirectional NLI entailment로 semantic cluster로 묶은 뒤, sample sequence log-likelihood를 cluster 단위로 log-sum-exp 집계해 cluster probability mass를 만들고 그 위에서 entropy를 계산한다. 산출물에는 \texttt{semantic\_entropy\_nli\_likelihood}, \texttt{semantic\_entropy\_cluster\_count}, \texttt{semantic\_entropy\_discrete\_cluster\_entropy}, \texttt{nli\_model\_ref}, sample/cluster log-likelihood가 함께 기록된다. SE는 prompt-level 특징이고, paired candidate setting에서 정답 후보와 환각 후보는 동일한 SE 값을 broadcast받는다.

\paragraph{Semantic Energy 및 logit diagnostics.}
Paper-faithful Semantic Energy는 Ma 2025의 Eq. (11)--(14)를 그대로 따른다. 동일한 N=10 답변과 S4의 NLI semantic cluster를 재사용하며, cluster를 다시 계산하지 않고 \texttt{(prompt\_id, sample\_index)}로 join한다. 식은 다음과 같다.
\begin{align}
\tilde{E}(x_t^{(i)}) &= -z_\theta(x_t^{(i)}) \quad \text{(Eq.~13: token energy = $-$selected\_token\_logit)} \\
E(x^{(i)}) &= \frac{1}{T_i}\sum_{t=1}^{T_i} \tilde{E}(x_t^{(i)}) \quad \text{(Eq.~11: per-response sequence-level energy)} \\
E_{\text{Bolt}}(\mathbb{C}_k) &= \sum_{x^{(i)} \in \mathbb{C}_k} E(x^{(i)}) \quad \text{(Eq.~12: cluster total energy, SUM)} \\
U(\mathbf{x}) &= \sum_{k=1}^{K} p(\mathbb{C}_k)\, E_{\text{Bolt}}(\mathbb{C}_k)
\end{align}
여기서 $p(\mathbb{C}_k) = \sum_{x^{(i)} \in \mathbb{C}_k} \bar{p}(x^{(i)} \mid q)$ 는 Eq.~(8)의 likelihood-based cluster probability이며, S4 Semantic Entropy 단계에서 산출된 값을 그대로 상속한다. 이는 Ma 2025 Section~2.2가 인용하는 Farquhar 2024의 cluster-likelihood weighting과 일치한다. cluster size가 큰 cluster는 Eq.~(12)의 SUM 정의에 의해 더 큰 cluster total energy를 갖고, 이를 likelihood-weighted aggregation에 반영한다. 부호 규약은 lower raw Energy = more reliable (higher Energy = higher uncertainty) 이다. 산출물 schema는 \texttt{semantic\_energy\_cluster\_uncertainty}와 prompt-level 보조 통계 \texttt{semantic\_energy\_sample\_energy} (= $\frac{1}{n}\sum_i E(x^{(i)})$, Eq.~14의 단순 collapsed reading), cluster id, selected-token logit/prob provenance, truncation/end-token policy, model/tokenizer ref를 포함한다.

한편 teacher-forced candidate scoring에서 얻은 token-level $-\log Z$ 평균, 평균 negative log probability, logit variance, confidence margin은 \texttt{semantic\_energy\_boltzmann\_diagnostic} 등 candidate-level Boltzmann-style energy diagnostic으로 별도 보존하며, paper-faithful Semantic Energy와 같은 column으로 섞지 않는다.

\paragraph{Corpus-grounded 특징.}
Corpus 특징은 candidate 답변 텍스트에서 entity를 추출한 뒤, Infini-gram-compatible count backend로 entity frequency와 entity-pair co-occurrence를 query하여 raw count, log-transformed continuous axis score, 3-bin primary \texttt{corpus\_axis\_bin}과 5-bin sensitivity \texttt{corpus\_axis\_bin\_5}, 그리고 backend/index/query/cache provenance를 함께 기록한다. 본 실험의 primary backend는 \texttt{InfinigramLocalEngineBackend} 위의 \texttt{v4\_dolmasample\_olmo} 인덱스 (16B Dolma sample tokens, OLMo tokenizer) 이며, 사전에 \texttt{s3://infini-gram-lite/index/v4\_dolmasample\_olmo/} 에서 로컬 디스크로 동기화한 뒤 \texttt{InfiniGramEngine.count\_cnf} AND query로 pair count를 얻는다. 공개 REST API (\texttt{https://api.infini-gram.io/}) 는 동일 cache schema의 fallback으로 둔다. 이 특징들은 외부 corpus에서 후보 답변의 지지 가능성을 간접적으로 나타내는 continuous corpus-support axis이며, hidden-state probe나 정답성 라벨을 사용하지 않는다. Low frequency와 zero co-occurrence는 hallucination label이 아니라 metric reliability를 조건화하는 binning 변수이다.

\subsection{실험 단계}

전체 실험은 다음 단계로 구성된다.

\begin{enumerate}
    \item \textbf{Paired dataset 생성}: TruthfulQA와 HaluEval-QA에서 prompt별 정답 후보와 환각 후보를 구성한다.
    \item \textbf{모델 출력 수집}: 각 prompt에 대해 N=10 answer-only free samples를 생성하고, 각 candidate 답변에 대해 teacher-forced full-logit scoring을 수행한다.
    \item \textbf{정답성 라벨 데이터 생성}: 데이터셋 annotation만 사용하여 정답 후보와 환각 후보의 정답성 라벨을 확정한다.
    \item \textbf{Semantic Entropy 계산}: prompt-level N=10 free samples를 NLI semantic cluster로 묶고 likelihood-based SE와 cluster metadata를 계산한다.
    \item \textbf{Corpus axis 계산}: candidate 답변에서 entity frequency와 entity-pair co-occurrence를 continuous axis와 bin으로 계산한다.
    \item \textbf{Semantic Energy 계산}: generated answers의 semantic clusters와 selected-token logits에서 paper-faithful Semantic Energy를 계산하고, candidate-level logit diagnostics를 별도 계산한다.
    \item \textbf{Feature table 결합}: prompt-level SE를 두 candidate row에 broadcast하고, candidate-level Energy/corpus 특징을 candidate id 기준으로 결합한다.
    \item \textbf{Fusion 및 baseline 평가}: SE-only, Energy-only, logit-diagnostic-only, corpus-axis-only, global fusion, corpus-bin-aware fusion, axis-interaction fusion을 비교한다.
    \item \textbf{Robustness 분석}: prompt 단위 bootstrap, leave-one-dataset-out, corpus-bin metric reliability, binning sensitivity를 수행한다.
\end{enumerate}

\subsection{평가 지표와 해석 원칙}

공통 baseline table에는 AUROC, AUPRC와 F1 등 표준 분류 지표를 보고한다. 학습 target은 데이터셋 annotation에서 직접 온 \texttt{is\_hallucination} (이진) 이다. 본 논문의 main 분석 축은 corpus-axis bin (3-bin primary, 5-bin / 10-bin sensitivity) 이며, 각 bin 안에서 SE/Energy/logit 진단의 AUROC와 paired matched-pair win rate를 보고한다. 다만 이 candidate-row aggregate metric은 모든 특징군에 대해 같은 의미를 갖지 않는다. Semantic Entropy는 prompt-level 특징이기 때문에, 같은 prompt의 정답 후보와 환각 후보가 동일한 SE 값을 가진다. 따라서 SE-only candidate-row AUROC가 0.5에 가까운 것은 SE가 무의미하다는 뜻이 아니라, prompt-level 특징을 candidate-level paired row ranking으로 평가한 구조적 결과이다.

Semantic Energy와 logit diagnostics는 candidate-level 또는 cluster-level 신호이지만, 그 신뢰도는 corpus support 조건에 따라 달라질 수 있다. 따라서 핵심 평가는 각 corpus-axis bin에서 환각 후보가 paired correct 후보보다 더 높은 risk를 받는지 확인하는 matched-pair metric이다. 이를 위해 hallucinated-minus-normal paired delta, pairwise win rate, prompt-grouped bootstrap confidence interval, leave-one-dataset-out, threshold/calibration sensitivity, 3-bin vs 5-bin corpus axis sensitivity를 모두 보고한다. Train/test split은 prompt 단위로 묶어 같은 prompt의 두 candidate row가 fold를 가로지르지 않도록 강제한다.

\subsection{재현성과 산출물 계약}

본 실험은 \texttt{run\_pipeline.py}의 S0--S9 단일 orchestrator로 실행되며, 매 stage마다 command, expected input/output, validator command, planned stdout/stderr log, git commit, Python 버전, platform, UTC timestamp가 manifest와 \texttt{script\_execution\_log}에 기록된다. 산출물 schema는 명시적으로 버전 표기 (예: \texttt{semantic\_entropy\_nli\_likelihood\_v1}, \texttt{semantic\_energy\_cluster\_v1}, \texttt{corpus\_axis\_counts\_v1}) 되어 schema mismatch가 있는 기존 artifact는 resume-skip되지 않는다. 모든 final artifact는 dataset annotation 라벨을 그대로 사용하며, heuristic 문자열 매칭이나 LLM-as-judge fallback은 thesis-valid label source로 인정하지 않는다. Final feature table은 11{,}630개 candidate row에 대해 N=10 NLI likelihood SE, paper-faithful Semantic Energy, candidate-level diagnostics, corpus axis bin과 backend provenance를 모두 포함하고, train fold만으로 fit된 condition-aware fusion이 held-out fold에서 평가된다.
