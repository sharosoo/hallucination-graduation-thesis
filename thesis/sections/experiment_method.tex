\section{실험 설정}

본 절에서는 본 연구에서 사용한 환각 탐지 실험 절차를 설명한다. 실험의 핵심은 모델이 새 답변을 생성한 뒤 사후에 정답성을 판정하는 방식이 아니라, 데이터셋이 \emph{이미 정답 답변과 환각 답변을 명시적으로 구분해 제공}하는 점을 활용하여 두 후보를 같은 질문 아래에서 \emph{쌍 비교(paired)}로 평가하는 것이다. 이로써 (i) 모델이 새로 만든 답을 정답이라고 판정하기 위한 문자열 일치 휴리스틱, (ii) 별도 LLM에게 정오를 묻는 LLM-as-judge 방식에 의존하지 않고, 이미 데이터셋 주석(annotation)으로 구분된 후보 쌍을 대상으로 불확실성 및 말뭉치 기반 특징이 환각 탐지에 어떤 신호를 제공하는지 분석할 수 있다.

\subsection{데이터셋 구성}

실험에는 TruthfulQA와 HaluEval-QA 두 데이터셋을 사용하였다. 두 데이터셋은 질문에 대해 정답 후보와 잘못된(또는 환각) 후보를 명시적으로 제공하므로 쌍 비교 평가 절차에 적합하다. TruthfulQA에서는 각 질문의 \texttt{correct\_answers}와 \texttt{incorrect\_answers}에서 하나의 정답 후보와 하나의 오답 후보를 결정론적으로 선택하였다. HaluEval-QA에서는 데이터셋이 제공하는 \texttt{right\_answer}와 \texttt{hallucinated\_answer}를 그대로 사용하였다.

각 질문은 하나의 프롬프트(prompt) 단위로 취급되며, 각 프롬프트에서 정확히 두 개의 평가 행(row)을 만든다. 하나는 정답 후보이고, 다른 하나는 환각 후보이다. 최종 실험 데이터는 5,815개 프롬프트와 11,630개 후보 행으로 구성된다. HaluEval-QA는 5,000개 프롬프트와 10,000개 행을, TruthfulQA는 정제 후 815개 프롬프트와 1,630개 행을 제공한다.

\subsection{정답성 라벨링}

정답성 라벨은 모델 출력이나 후처리 judge에서 오지 않는다. 각 후보 행의 주 라벨은 데이터셋 주석이 제공하는 환각 여부(\texttt{is\_hallucination}) 이진 라벨이며, 이것이 최종 환각 탐지 평가의 기준이다. 즉 본 연구는 데이터셋 주석을 \emph{근거 진실(ground truth)} 로 받아들이고, 모델 출력 자체에서 라벨을 유도하지 않는다. 이때 Semantic Entropy는 프롬프트 단위 값이므로 같은 질문의 정답 후보와 환각 후보가 동일한 SE 값을 공유한다.

\subsection{모델 점수화 절차}

모델은 Qwen2.5-3B 계열 인과 언어모델(causal LM)을 사용하였고, 같은 계열 토크나이저(어휘 크기 151{,}936)의 전체 어휘 로짓(full-vocabulary logits)을 그대로 보존하기 위해 생성 산출물은 \texttt{float16} sidecar parquet에 저장한다. 실험은 두 종류의 모델 출력을 사용한다.

\begin{itemize}
    \item \textbf{자유 표본 추출 (free sampling)}: Semantic Entropy 계산을 위해 각 프롬프트에 대해 \emph{답변만 생성} 절차(answer-only protocol)로 N=10개의 짧은 답변을 표본 추출한다. 이는 프롬프트 단위 신호의 입력이다.
    \item \textbf{교사 강제 점수화 (teacher-forced scoring)}: 후보 단위 특징을 계산하기 위해 각 정답/환각 후보를 교사 강제 방식으로 점수화한다. 즉 모델이 후보 답을 새로 생성하는 것이 아니라, \emph{주어진 후보 답의 각 토큰 위치}에서 전체 어휘 로짓, 토큰별 \texttt{logsumexp}(분배 함수 $-\log Z$), 선택된 토큰의 로짓과 로그 확률을 모두 기록한다.
\end{itemize}

이 구분은 이후 평가 해석에서 중요하다. Semantic Entropy는 같은 프롬프트의 두 후보 행에 동일 값으로 복사되지만, 후보 단위 진단(평균 음의 로그 확률, 로짓 분산, confidence margin)은 후보마다 다른 값을 가지며, 원논문 정의의 Semantic Energy는 의미 클러스터 단위로 정의된다.

답변만 생성 절차는 \texttt{generation.max\_new\_tokens=96}, \texttt{answer\_only.max\_answer\_tokens=64}, \texttt{answer\_only.max\_invalid\_attempts=32}, \texttt{fail\_on\_max\_new\_tokens=true}로 고정한다. 이는 HaluEval-QA의 긴 문맥 일부 프롬프트가 짧은 토큰 한도(32/64/8)에서 표본 재시도를 모두 소진해 단계가 중단되는 현상을 해결하기 위한 값이며, 짧은 답변 절차의 의도는 유지하되 다중 개체 답변을 안전하게 표현할 여유를 둔다.

\subsection{특징 계산}

본 연구는 세 종류의 특징군을 사용한다.

\paragraph{Semantic Entropy.}
Semantic Entropy는 한 프롬프트에서 얻은 N=10 답변을 \texttt{microsoft/deberta-large-mnli} 기반 양방향 자연어 추론(NLI) 함의 관계로 의미 클러스터로 묶은 뒤, 표본 시퀀스 로그 가능도를 클러스터 단위로 log-sum-exp 집계해 클러스터 확률 질량을 만들고 그 위에서 엔트로피를 계산한다. 산출물에는 \texttt{semantic\_entropy\_nli\_likelihood}, \texttt{semantic\_entropy\_cluster\_count}, NLI 모델 참조, 표본/클러스터 로그 가능도가 함께 기록된다. SE는 프롬프트 단위 특징이고, 쌍 비교 후보 설정에서 정답 후보와 환각 후보는 동일한 SE 값을 받는다.

\paragraph{Semantic Energy 및 후보 단위 로짓 진단.}
원논문 정의의 Semantic Energy는 Ma 등(2025)의 식 (11)--(14)를 그대로 따른다. 동일한 N=10 답변과 SE 단계의 NLI 의미 클러스터를 재사용하며, 클러스터를 다시 계산하지 않고 \texttt{(prompt\_id, sample\_index)}로 join한다. 식은 다음과 같다.
\begin{align}
\tilde{E}(x_t^{(i)}) &= -z_\theta(x_t^{(i)}) \quad \text{(식 13: 토큰 에너지 = $-$ 선택 토큰 로짓)} \\
E(x^{(i)}) &= \frac{1}{T_i}\sum_{t=1}^{T_i} \tilde{E}(x_t^{(i)}) \quad \text{(식 11: 표본별 시퀀스 단위 에너지)} \\
E_{\text{Bolt}}(\mathbb{C}_k) &= \sum_{x^{(i)} \in \mathbb{C}_k} E(x^{(i)}) \quad \text{(식 12: 클러스터 합계 에너지)} \\
U(\mathbf{x}) &= \sum_{k=1}^{K} p(\mathbb{C}_k)\, E_{\text{Bolt}}(\mathbb{C}_k) \quad \text{(식 14: 최종 불확실성)}
\end{align}
여기서 $p(\mathbb{C}_k) = \sum_{x^{(i)} \in \mathbb{C}_k} \bar{p}(x^{(i)} \mid q)$는 식 (8)의 가능도 기반 클러스터 확률이며, SE 단계에서 산출된 값을 그대로 상속한다. 이는 Ma 등(2025) §2.2가 인용하는 Farquhar 등(2024)의 클러스터 가능도 가중과 일치한다. 부호 규약은 \emph{낮은 에너지 = 높은 신뢰성}이다(높은 에너지 = 높은 불확실성).

한편 교사 강제 후보 점수화에서 얻은 토큰 단위 $-\log Z$ 평균, 평균 음의 로그 확률(NLL), 로짓 분산, confidence margin은 \emph{후보 단위 볼츠만형 에너지 진단(Boltzmann-style energy diagnostic)}으로 별도 보존하며, 원논문 정의의 Semantic Energy와 같은 열에 합치지 않는다.

\paragraph{말뭉치 기반 특징.}
말뭉치 특징은 후보 답변 텍스트에서 개체(entity)를 추출한 뒤 Infini-gram 호환 카운트 backend로 \emph{개체 빈도(entity frequency)}와 \emph{개체쌍 공기 빈도(entity-pair co-occurrence)}를 질의하여, 원시 카운트, 로그 변환 연속 점수, 그리고 3-bin/5-bin/10-bin 지지도 구간 라벨(\texttt{corpus\_axis\_bin}, \texttt{\_5}, \texttt{\_10})을 함께 기록한다. 본 실험의 기본 backend는 \texttt{InfinigramLocalEngineBackend} 위의 \texttt{v4\_dolmasample\_olmo} 인덱스(16B Dolma 샘플 토큰, OLMo 토크나이저)이며, 사전에 \texttt{s3://infini-gram-lite/index/v4\_dolmasample\_olmo/} 에서 로컬 디스크로 동기화한 뒤 \texttt{InfiniGramEngine.count\_cnf} AND 질의로 쌍 카운트를 얻는다. 공개 REST API(\texttt{https://api.infini-gram.io/})는 동일 캐시 스키마의 fallback으로 둔다. 이 특징들은 외부 말뭉치에서 후보 답변의 \emph{지지 가능성을 간접적으로 나타내는 보조 신호}이며, 모델의 은닉 상태(hidden-state)나 정답성 라벨을 사용하지 않는다. 낮은 빈도와 0 공기 빈도는 환각 라벨이 아니라 \emph{지표 신뢰성을 조건화하는 구간 변수}이다.

\subsection{실험 단계 (S0--S9)}

전체 실험은 \texttt{run\_pipeline.py}가 자동으로 실행하는 9단계(S0부터 S9)로 구성된다. 각 단계는 입력/출력 파일이 명시되어 있어 단일 단계 재실행, 결과 산출물 검증, 그리고 동일 입력에서의 결정론적 재현이 가능하다.

\begin{enumerate}
    \item[\textbf{S0--S1.}] \textbf{쌍 비교 데이터셋 구성}: TruthfulQA와 HaluEval-QA에서 프롬프트별 정답 후보와 환각 후보를 구성한다.
    \item[\textbf{S2--S3.}] \textbf{모델 출력 수집}: 각 프롬프트에 대해 N=10 답변을 자유 표본 추출하고, 각 후보 답변에 대해 교사 강제 전체 어휘 점수화를 수행한다. 정답성 라벨은 데이터셋 주석만 사용해 확정한다.
    \item[\textbf{S4.}] \textbf{Semantic Entropy 계산}: 프롬프트 단위 N=10 표본을 NLI 의미 클러스터로 묶고 가능도 기반 SE와 클러스터 메타데이터를 계산한다.
    \item[\textbf{S5.}] \textbf{말뭉치 지지도 축 계산}: 후보 답변에서 개체 빈도와 개체쌍 공기 빈도를 연속 점수와 구간 라벨(3-bin / 5-bin / 10-bin)로 계산한다.
    \item[\textbf{S6.}] \textbf{Semantic Energy 계산}: 생성된 답변의 의미 클러스터와 선택 토큰 로짓에서 원논문 정의 Semantic Energy를 계산하고, 후보 단위 로짓 진단을 별도 계산한다.
    \item[\textbf{S7.}] \textbf{특징 테이블 결합}: 프롬프트 단위 SE를 두 후보 행에 복사하고, 후보 단위 Energy/말뭉치 특징을 후보 id 기준으로 결합한다.
    \item[\textbf{S8.}] \textbf{결합 및 베이스라인 평가}: SE 단일, Energy 단일, 로짓 진단 단일, 말뭉치 신호 단일, 전역 학습 결합(말뭉치 미사용/사용), 구간별 가중 결합, 축 상호작용 결합, 사이킷런 비선형 결합기들을 비교한다.
    \item[\textbf{S9.}] \textbf{Robustness 분석}: 프롬프트 단위 부트스트랩, leave-one-dataset-out, 구간별 신뢰성, 구간 분할 민감도(3/5/10-bin), 임계값/보정 진단을 수행한다.
\end{enumerate}

\subsection{평가 지표와 해석 원칙}

공통 베이스라인 표에는 AUROC, AUPRC, F1 등 표준 분류 지표를 보고한다. 학습 라벨은 데이터셋 주석에서 직접 온 환각 여부(\texttt{is\_hallucination}) 이진 라벨이다. 본 논문의 주 분석 축은 말뭉치 지지도 구간(3-bin 기본, 5-bin / 10-bin 민감도)이며, 각 구간 안에서 SE/Energy/로짓 진단의 AUROC와 쌍 비교 승률을 보고한다.

다만 이 후보 행 단위 종합 지표는 모든 특징군에 대해 같은 의미를 갖지 않는다. Semantic Entropy는 프롬프트 단위 특징이기 때문에, 같은 프롬프트의 정답 후보와 환각 후보가 동일한 SE 값을 가진다. 따라서 SE 단일의 후보 행 단위 AUROC가 0.5에 묶이는 것은 \emph{SE가 무의미하다는 뜻이 아니라, 프롬프트 단위 특징을 후보 단위 순위 평가에 적용한 구조적 결과}이다.

Semantic Energy와 로짓 진단은 후보 단위 또는 클러스터 단위 신호이지만, 그 신뢰도는 말뭉치 지지도 조건에 따라 달라질 수 있다. 따라서 핵심 평가는 각 지지도 구간에서 환각 후보가 짝지어진 정답 후보보다 더 높은 위험 점수를 받는지 확인하는 \emph{쌍 비교 지표(matched-pair metric)}이다. 이를 위해 (i) 환각 후보 점수 $-$ 정답 후보 점수의 평균 차이(쌍 비교 평균 차), (ii) 쌍 비교 승률(환각이 더 높은 위험을 받는 비율), (iii) 프롬프트 단위 부트스트랩 95\% 신뢰구간, (iv) leave-one-dataset-out 일반화, (v) 임계값/보정 민감도, (vi) 3/5/10-bin 구간 분할 민감도를 모두 보고한다. 학습/평가 분할은 프롬프트 단위로 묶어 같은 프롬프트의 두 후보 행이 fold를 가로지르지 않도록 강제한다.

\subsection{재현 가능성}

본 실험은 \texttt{run\_pipeline.py}가 자동 실행하는 9단계(S0--S9)로 진행되며, 각 단계마다 실행 명령, 기대되는 입출력 파일, 검증 명령, git 커밋 해시, Python 버전, 플랫폼, UTC 시각이 manifest와 실행 로그에 기록된다. 산출물 스키마는 명시적으로 버전 태그(예: \texttt{semantic\_entropy\_nli\_likelihood\_v1}, \texttt{semantic\_energy\_cluster\_v1}, \texttt{corpus\_axis\_counts\_v1})를 갖도록 하여, 스키마가 다른 기존 산출물은 ``재실행 시 건너뛰기'' 대상에서 제외된다. 모든 최종 산출물은 데이터셋 주석 라벨을 그대로 사용하며, 휴리스틱 문자열 일치나 LLM-as-judge 기반 정오 판정은 본 연구의 라벨 출처로 인정하지 않는다. 최종 특징 테이블은 11{,}630개 후보 행에 대해 N=10 NLI 기반 SE, 원논문 정의 Semantic Energy, 후보 단위 로짓 진단, 말뭉치 지지도 구간과 backend 출처 정보를 모두 포함하며, 학습 fold만으로 적합된 조건부 결합이 평가 fold에서 검증된다.
