\section{실험 설정}

본 절에서는 현재 저장소의 \texttt{experiments/} 산출물을 기준으로 평가 절차를 설명한다. 이전 초안의 TruthfulQA 단일 설정과 SE 선택 규칙은 최종 주장으로 사용하지 않고, 현재 실험명인 \textit{Corpus-Grounded Selective Fusion Detector}와 그 기준선 비교를 중심으로 정리한다.

\subsection{데이터셋과 표본}

현재 feature table은 5개 데이터셋에서 총 1000개 행을 포함한다. 데이터셋은 HaluEval-QA, HaluEval-dialogue, Natural Questions, TriviaQA, TruthfulQA이며, 평가 산출물은 데이터셋별 가시성을 유지하기 위해 leave-one-dataset-out 방식으로 집계된다. 전체 레이블 수는 정상 402개, high-diversity hallucination 481개, low-diversity hallucination 117개, ambiguous incorrect 0개이다. 현재 행에는 \texttt{AMBIGUOUS\_INCORRECT}가 없으므로, 회색지대 성능이나 임계값 민감도 주장은 새로 만들지 않는다.

\subsection{실험 파이프라인}

현재 파이프라인은 저장소 내부의 \texttt{experiments/} 코드와 uv-managed 환경을 기준으로 실행된다. 검증 명령은 전역 인터프리터나 임의의 package 설치가 아니라 \verb|uv run|으로 실행한다. 이전 산출물에는 외부 실험 디렉터리에서 온 캐시와 프록시가 포함되어 있지만, 향후 논문 유효 실행 경로는 repo-owned generation을 사용해야 한다. 그 실행 경로는 행 단위 \texttt{full logits}, per-step \texttt{logsumexp}, 생성 토큰 id, 모델 및 토크나이저 provenance를 보존해야 한다.

\subsubsection{지표와 특징}

Semantic Entropy는 반복 응답의 의미 클러스터 분포에서 계산한 신호로 유지한다. Corpus feature는 QuCo-RAG-style entity frequency, entity-pair co-occurrence, low-frequency flag를 외부 corpus-grounded proxy로 사용한다. 이 corpus 신호는 hidden-state probe가 아니며, Phillips et al.은 선택 예측과 learned combination의 개념적 배경으로만 사용한다. \texttt{experiments/literature/evidence\_notes/quco\_vs\_probe.md}는 이 구분을 보존하기 위한 evidence note이다.

Semantic Energy는 Ma 등의 preprint에서 제안된 low-diversity 또는 single-cluster 상황의 동기 신호로만 다룬다. 현재 저장소의 산출물은 true Boltzmann Energy 계산에 필요한 행 단위 full logits를 포함하지 않는다. 따라서 \verb|semantic_energy_proxy|는 진단 메타데이터일 뿐이며, 논문 성능 주장에는 쓰지 않는다. \texttt{experiments/literature/evidence\_notes/semantic\_energy\_single\_cluster.md}도 Zero-SE 숫자를 직접 인용하지 말고 single-cluster 문헌 동기로만 쓰라는 guardrail을 둔다.

\subsubsection{평가 기준}

탐지 성능은 AUROC, AUPRC, accuracy, precision, recall, F1로 보고한다. Energy-only, fixed linear SE/Energy 조합, hard cascade는 모두 true Boltzmann Energy가 필요하므로 현재 결과에서는 unavailable로 남긴다. learned fusion with corpus는 corpus feature를 직접 학습 특징으로 추가한 방법이며, corpus-risk-only는 제안 방법이 아니라 corpus feature만 보는 기준선이다.

\subsection{재현성}

현재 논문 문장은 exported JSON summary와 table에서 값을 가져온다. 파일명은 \path{thesis_evidence_summary.json}, \path{thesis_evidence_table.tex}이다. 두 파일은 \verb|uv run|으로 실행하는 export script로 생성된다. evidence note와 thesis 연결도 \verb|uv run| 기반 validator로 검증한다.
