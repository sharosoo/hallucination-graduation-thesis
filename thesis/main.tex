\begin{abstract}
대규모 언어모델(LLM)의 환각(hallucination) 문제는 실용적 적용에서 중요한 장애물이다. 본 논문은 반복 샘플링 기반 Semantic Entropy(SE), Ma 등의 Semantic Energy 동기, 그리고 QuCo-RAG-style corpus statistics를 바탕으로 LLM 응답의 환각 탐지 신호를 검토한다.

최종 실험 주장은 현재 저장소에서 생성된 type analysis, fusion, robustness 산출물에 한정한다. 현재 산출물에서 SE-only는 aggregate AUROC 0.675244를 보였고, corpus-risk-only는 0.499133, learned fusion without corpus는 0.477294, learned fusion with corpus는 0.444207을 보였다. 즉 현재 관측 결과는 corpus feature를 추가한 learned fusion이 AUROC 기준으로 SE-only와 learned fusion without corpus보다 낮다는 점을 보여주며, corpus-fusion 성능 향상을 주장하지 않는다.

또한 true Boltzmann Energy, Energy-only, fixed SE/Energy 조합, hard cascade는 현재 사용할 수 없다. 현재 row-level 산출물에는 행 단위 full logits가 보존되어 있지 않으므로, \verb|semantic_energy_proxy|는 thesis-valid evidence가 아니다. 향후 유효한 Energy 평가는 uv-managed repo-owned generation으로 생성된 row-level full logits, per-step logsumexp, token id, provenance를 보존한 뒤 다시 수행해야 한다.

본 논문의 기여는 유리한 결과를 과장하는 데 있지 않다. 현재 evidence export를 통해 사용 가능한 결과와 unavailable baseline을 분리하고, 문헌 기반 동기와 repo-owned 재현성 요구사항을 명확히 하여 후속 실험의 기준을 제시한다.
\end{abstract}

\chapter{서론}

\section{연구 배경}

대규모 언어모델(Large Language Model, LLM)은 텍스트 생성, 질의응답, 요약, 번역 등 다양한 자연어 처리 태스크에서 널리 사용되고 있다. 그러나 최신 LLM도 사실과 다른 내용을 그럴듯하게 생성하는 환각(hallucination) 문제를 보이며, 이는 LLM 신뢰성과 안전성의 핵심 과제로 논의되어 왔다\cite{huang2025survey}. 생성 텍스트의 신뢰성은 사실성(factuality)과 충실성(faithfulness) 관점에서 구분해 다룰 수 있으며\cite{maynez2020faithfulness}, 본 연구는 외부 사실과 충돌하거나 검증 가능한 정답과 맞지 않는 응답을 탐지하는 문제에 초점을 둔다.

환각 문제는 LLM의 실용적 적용에서 중요한 장애물이다. 의료 분야에서 잘못된 진단 정보를 제공하거나, 법률 분야에서 존재하지 않는 판례를 인용하거나, 금융 분야에서 부정확한 수치를 제시하는 경우 큰 피해를 초래할 수 있다. TruthfulQA는 모델이 인간이 흔히 믿는 거짓 정보를 얼마나 그럴듯하게 모방하는지 평가하기 위해 제안된 벤치마크이며\cite{truthfulqa}, 이러한 설정은 LLM 응답을 사후에 검증하고 환각을 자동으로 탐지하는 방법의 필요성을 보여준다.

환각 탐지를 위한 대표적인 불확실성 기반 방법으로 Semantic Entropy(SE)가 있다. SE는 하나의 질문에 대해 여러 응답을 샘플링한 뒤, 자연어 추론(Natural Language Inference, NLI) 모델을 사용하여 의미적으로 클러스터링하고, 클러스터 분포의 엔트로피를 계산한다\cite{farquhar2024}. 반복 샘플링과 응답 간 일관성은 SelfCheckGPT와 같은 선행 연구에서도 환각 탐지를 위한 중요한 단서로 사용되었다\cite{selfcheckgpt2023}. SE가 높으면 모델 응답이 여러 의미로 갈라지므로 의미적 불확실성이 크다는 신호가 된다.

그러나 SE는 응답 간 의미적 다양성에 의존하므로, 모델이 모든 응답에서 같은 의미의 틀린 답변을 반복하면 판별력이 약해진다. 예를 들어 같은 질문을 다섯 번 샘플링했는데 다섯 응답이 모두 동일한 잘못된 인물명이나 수치를 답한다면, 응답들은 의미적으로 완전히 일관되어 하나의 클러스터에 모인다. 이때 SE는 0에 가까워지지만, 낮은 SE가 곧 정답을 의미하지는 않는다. 단지 ``서로 다른 답을 내지 않았다''는 사실만 알려줄 뿐이다. 본 연구는 이처럼 모든 응답이 하나의 의미 클러스터에 속해 SE가 환각 여부를 구분하기 어려워지는 현상을 \textbf{Zero-SE 문제}라고 정의한다.

Zero-SE 설정에서는 ``응답들이 서로 같은가''라는 질문과 ``그 같은 응답이 사실인가''라는 질문이 서로 다른 문제로 분리된다. 탐지 과정에서는 먼저 SE가 판별 정보를 제공하는 상황인지 확인해야 한다. 응답들이 여러 의미로 갈라지면 SE를 의미적 불확실성 신호로 사용할 수 있고, 모든 응답이 하나의 의미로 수렴하면 SE만으로는 사실성 판단이 남는다. 본 연구는 이 문제를 현재 evidence 범위에서는 SE와 corpus-grounded learned fusion 결과로 정리하고, Semantic Energy는 row-level full logits를 확보한 뒤 재검증해야 할 보조 신호 후보로만 둔다.


\section{연구 목적}

본 연구의 목적은 다음과 같다:

\begin{enumerate}
    \item \textbf{현재 evidence의 정리}: 현재 저장소가 생성한 type analysis, fusion, robustness 결과만 최종 실험 근거로 사용한다.
    \item \textbf{결합 방법의 검증}: \textit{Corpus-Grounded Selective Fusion Detector}가 SE-only, corpus-risk-only, learned fusion without corpus와 비교해 어떤 성능을 보이는지 확인한다.
    \item \textbf{Energy 한계의 명시}: true Boltzmann Energy와 Energy-dependent baselines가 row-level full logits 없이는 사용할 수 없음을 분리해 보고한다.
    \item \textbf{재현성 경로의 고정}: 향후 실험은 \verb|uv run|과 repo-owned generation을 사용하며, full logits preservation을 필수 조건으로 둔다.
\end{enumerate}

\section{환각 분류, 사실성, 벤치마크}

환각은 LLM이 입력, 문맥, 또는 외부 사실과 맞지 않는 내용을 생성하는 현상으로 다루어진다. Huang 등은 LLM 환각의 원인, 유형, 탐지 과제를 폭넓게 정리하며, 환각 탐지가 LLM 신뢰성 확보를 위한 핵심 문제임을 보인다\cite{huang2025survey}. Maynez 등은 생성 텍스트 평가에서 사실성(factuality)과 충실성(faithfulness)을 구분해 논의했으며\cite{maynez2020faithfulness}, 이 구분은 환각 탐지 문장을 작성할 때 단순한 ``틀림''보다 어떤 기준에서 맞지 않는지 명확히 해야 함을 보여준다.

벤치마크와 평가 연구는 환각 탐지의 대상과 평가 기준을 제공한다. TruthfulQA는 모델이 인간의 오해나 거짓 믿음을 모방하는지를 측정하는 질의응답 벤치마크이다\cite{truthfulqa}. FActScore는 장문 생성에서 원자적 사실 단위의 정밀도를 평가하는 방법을 제안했으며\cite{factscore2023}, AlignScore는 생성 내용과 근거 사이의 정렬을 통해 사실 일관성을 평가한다\cite{alignscore2023}. 이러한 연구들은 환각을 평가하고 분류하는 기준을 제공하지만, 본 연구의 중심인 Zero-SE 설정에서 SE와 Energy를 어떻게 결합할지는 직접 다루지 않는다.

\section{불확실성 및 반복 샘플링 기반 탐지}

SelfCheckGPT는 외부 지식 베이스 없이 여러 응답을 샘플링하고 응답 간 일관성을 비교해 환각을 탐지하는 접근을 제안했다\cite{selfcheckgpt2023}. 이 흐름은 하나의 응답만 보는 대신, 같은 질문에 대한 여러 생성 결과의 분산과 일관성을 환각 탐지 신호로 본다. Semantic Entropy도 이 관점과 연결되지만, 응답을 의미 클러스터로 묶고 클러스터 분포의 엔트로피를 사용한다는 점에서 더 명시적인 의미 단위 불확실성 측정 방법이다\cite{farquhar2024}.

\section{Semantic Entropy (SE)}

Semantic Entropy는 Farquhar 등이 2024년 Nature에 발표한 LLM 불확실성 측정 방법이다\cite{farquhar2024}. 기존의 토큰 단위 확률 기반 불확실성 측정 방법과 달리, SE는 응답의 의미적 내용을 기반으로 불확실성을 측정한다.

\subsection{작동 원리}

SE의 계산 과정은 다음과 같다:

\begin{enumerate}
    \item \textbf{응답 샘플링}: 하나의 질문 $q$에 대해 LLM으로부터 $K$개의 응답 $\{r_1, r_2, ..., r_K\}$를 샘플링한다.
    
    \item \textbf{의미적 클러스터링}: NLI 모델을 사용하여 응답들을 의미적으로 클러스터링한다. 두 응답이 서로 entailment 관계에 있으면 같은 클러스터로 분류한다.
    
    \item \textbf{엔트로피 계산}: 클러스터 분포의 Shannon entropy를 계산한다.
\end{enumerate}

수식으로 표현하면:
\begin{equation}
    SE = -\sum_{c \in C} p(c) \log p(c)
\end{equation}
여기서 $C$는 의미 클러스터 집합이고, $p(c)$는 클러스터 $c$에 속하는 응답의 비율이다.

\subsection{한계점}

SE의 주요 한계점은 다음과 같다:
\begin{itemize}
    \item \textbf{Hard Clustering}: 응답들을 이진적으로(동일/다름) 분류하여 세밀한 유사도를 반영하지 못한다.
    \item \textbf{Zero-SE 문제}: 모든 응답이 하나의 클러스터에 속하면 SE=0이 되어 판별력이 사라진다.
    \item \textbf{응답 길이와 클러스터링 의존성}: 긴 응답이나 복잡한 답변에서는 NLI 기반 클러스터링 품질에 따라 결과가 달라질 수 있다.
\end{itemize}


\section{SE 계열 확장 방법: KLE와 SNNE}

KLE와 SNNE는 SE의 한계를 완화하려는 후속 방법들이다. Nikitin 등(2024)의 KLE는 hard clustering 대신 의미 커널과 von Neumann Entropy를 사용해 SE를 더 일반적인 불확실성 프레임워크로 확장한다\cite{nikitin2024}. Nguyen 등(2025)의 SNNE는 명시적 클러스터링 없이 응답 간 pairwise semantic similarity를 사용해 긴 응답에서 클러스터 수가 불필요하게 증가하는 문제를 줄이려 한다\cite{nguyen2025}. 두 방법은 SE 계열 연구의 확장 방향을 보여주지만, 본 연구의 초점인 Zero-SE 구간에서 SE와 Energy를 어떻게 선택적으로 결합할지는 직접 다루지 않는다.


\section{Semantic Energy}

Ma 등(2025)이 제안한 Semantic Energy는 토큰 단위의 확신도를 측정하는 방법이다\cite{ma2025}. 현재 문헌 상태에서는 기존 방법 맥락으로만 사용하며, 본 연구의 핵심 주장은 실험에서 관찰한 SE와 Energy의 보완적 신호 차이에 둔다.

\subsection{작동 원리}

Energy는 LLM이 각 토큰을 생성할 때 부여한 \textbf{softmax 전 raw logit} 값을 사용한다:
\begin{equation}
    Energy = \frac{1}{nT} \sum_{i=1}^{n} \sum_{t=1}^{T_i} -z_\theta(x_t^{(i)})
\end{equation}
여기서 $z_\theta(x_t)$는 토큰 $x_t$의 logit 값이다.

\subsection{Zero-SE 보완}

Energy의 중요한 역할은 single-cluster 또는 낮은 다양성 상황에서 토큰 수준 점수 정보를 제공할 수 있다는 문헌적 동기이다. 그러나 현재 저장소 산출물은 true Boltzmann Energy 계산에 필요한 row-level full logits를 보존하지 않았으므로, 이 논문에서는 Energy 성능 수치를 최종 결과로 주장하지 않는다. Energy는 SE를 대체하는 실험 결과가 아니라, full logits 보존 후 재검증해야 할 동기 신호로 다룬다.


\section{기존 연구의 한계 및 연구 공백}

기존 연구들의 한계를 정리하면 다음과 같다:

\begin{enumerate}
    \item \textbf{환각 분류 및 평가 연구}: 환각의 정의, 사실성, 벤치마크를 제공하지만 Zero-SE 영역의 탐지 신호 결합 문제를 직접 다루지 않는다.
    \item \textbf{SE 계열 방법}: 반복 샘플링과 의미 클러스터 분포를 활용하지만, 모든 응답이 하나의 의미 클러스터에 모이는 경우 판별 정보가 부족해진다.
    \item \textbf{Energy 계열 점수}: 토큰 수준 확신도 정보를 제공하지만, 응답 간 의미적 다양성 정보를 직접 활용하지 못한다.
    \item \textbf{결합 구조의 필요성}: 단일 점수로 모든 환각 유형을 처리하기보다, SE가 유효한 구간과 Zero-SE 구간을 나누어 서로 다른 신호를 적용하는 구조가 필요하다.
\end{enumerate}

본 연구는 SE, Energy 동기, corpus-grounded signal의 결합 가능성을 검토하되, 현재 실험에서는 Energy-dependent baseline이 full logits 부족으로 unavailable임을 명시한다.

\section{환각이 드러나는 두 가지 양상}

Zero-SE 문제를 이해하려면, 환각이 반복 샘플링에서 서로 다른 방식으로 나타날 수 있음을 구분할 필요가 있다. 하나는 같은 질문에 대해 서로 다른 답변 후보가 나타나는 경우이다. 어떤 샘플에서는 A라고 답하고, 다른 샘플에서는 B라고 답한다면, 모델의 출력은 여러 의미 후보 사이에서 흔들리는 것처럼 보인다. 본 연구에서는 이런 양상을 혼란형으로 부른다. 다른 하나는 반복 샘플링 결과가 일관되지만, 그 답변 자체가 사실이 아니거나 근거 자료로 지지되지 않는 경우이다. 본 연구에서는 이런 양상을 지어냄형 또는 fabrication-like 사례로 구분한다.

이 구분은 문헌의 표준 분류명을 새로 제안하려는 것이 아니라, 기존 연구가 다룬 두 종류의 탐지 단서를 반복 샘플링 설정에 맞게 정리한 것이다. NLG와 LLM 환각 연구는 사실성(factuality), 충실성(faithfulness), 근거와의 일관성, 외부 지식으로 지지되는지 여부를 구분해 다루며\cite{ji2023survey,maynez2020faithfulness,pagnoni2021frank}, 사실성 평가 연구도 생성 문장을 원자적 사실 단위로 나누어 각 사실이 근거 자료에 의해 지지되는지를 평가한다\cite{factscore2023}. 따라서 본 연구에서 말하는 혼란과 지어냄은 모델의 숨은 의도나 내적 상태를 단정하는 표현이 아니라, 여러 번 생성한 출력에서 나타나는 응답 분포와 근거성의 차이를 설명하기 위한 구분이다.

반복 샘플링 기반 연구는 같은 질문에 대해 여러 응답을 생성했을 때 응답 간 불일치나 의미적 분산이 환각 탐지 단서가 될 수 있음을 보였다\cite{selfcheckgpt2023,farquhar2024}. 이는 혼란형 패턴과 잘 맞는다. 응답들이 서로 다른 의미 클러스터로 갈라지면 SE는 그 갈라짐을 엔트로피로 요약할 수 있다. 그러나 이러한 접근은 응답들이 모두 같은 의미로 모이는 경우에는 약해진다. 의미적으로 일관된 출력이 항상 사실적으로 옳다는 뜻은 아니기 때문이다. TruthfulQA는 모델이 그럴듯하지만 거짓인 답변을 낼 수 있음을 평가하고\cite{truthfulqa}, FActScore와 HALoGEN은 생성된 개별 사실이 외부 근거로 지지되는지 또는 fabrication에 해당하는지를 따로 검증한다\cite{factscore2023,ravichander2025halogen}. 즉, \textbf{응답 간 일관성}과 \textbf{정답 또는 근거에 대한 정확성}은 분리될 수 있다.

\subsection{혼란형 환각 (Confusion-like Pattern)}

\textbf{정의}: 같은 질문에 대해 서로 다른 의미의 응답 후보들이 경쟁하는 것으로 관측되는 경우이다. 예를 들어 반복 샘플링 결과가 서로 다른 인물명, 날짜, 원인을 답한다면, 모델 출력은 하나의 안정된 의미로 수렴하지 않는다. 이는 SelfCheckGPT가 활용한 샘플 간 불일치/일관성 비교와, Farquhar 등이 semantic entropy로 포착한 의미적 불확실성 신호와 맞닿아 있다\cite{selfcheckgpt2023,farquhar2024}.

\textbf{특성}:
\begin{itemize}
    \item 반복 샘플링 시 둘 이상의 의미 클러스터가 형성됨
    \item 응답 간 의미적 분산이 커서 정답/오답 후보가 함께 나타날 수 있음
    \item 여러 번 질문 시 \textbf{다양한 응답 패턴}이 관측됨 (상대적으로 높은 SE)
\end{itemize}

탐지 관점에서 SE는 응답 간 다양성을 요약하므로, 의미적 갈라짐이 존재하는 구간에서 우선적인 신호가 된다. 이 범주에서 SE를 사용하는 이유는 ``환각의 원인''을 아는 데 있지 않고, 샘플 간 의미적 불안정성이 관측된다는 점을 점수화하는 데 있다.

\subsection{지어냄형 환각 (Confabulation/Fabrication-like Pattern)}

\textbf{정의}: 반복 샘플링에서 응답이 거의 하나의 의미 패턴으로 수렴하지만, 그 패턴의 사실성 또는 근거 지지 여부는 별도 신호 없이는 판별하기 어려운 경우이다. 예를 들어 모델이 다섯 번 모두 같은 판례명, 논문 제목, 수치를 말하더라도 그것이 실제로 존재하지 않거나 근거 문서에 없는 내용이면 환각일 수 있다. 이때 핵심은 ``모델이 지어내려 했다''는 의도가 아니라, 일관된 출력이 외부 사실이나 근거 자료로 지지되지 않을 수 있다는 점이다. TruthfulQA는 그럴듯하지만 거짓인 답변을 평가 대상으로 삼고\cite{truthfulqa}, FActScore는 생성된 원자적 사실이 신뢰 가능한 근거에 의해 지지되는지를 평가하며\cite{factscore2023}, HALoGEN은 fabrication을 포함한 LLM 환각 오류 유형을 근거 자료와 대조해 분석한다\cite{ravichander2025halogen}.

\textbf{특성}:
\begin{itemize}
    \item 반복 샘플링 시 응답이 단일 의미 클러스터에 가깝게 모임
    \item 응답들이 서로 비슷하므로 응답 간 다양성만으로는 판별 정보가 줄어듦
    \item 여러 번 질문 시 \textbf{일관된 답변}이 반복될 수 있으나, 그 답변이 맞는지 틀린지는 사실성 평가나 토큰 수준 확신도 등 다른 신호가 필요함
\end{itemize}

탐지 관점에서 이 범주에서는 SE가 작아지거나 0이 되어 구분력이 약해질 수 있으므로, Energy와 같은 토큰/로그릿 기반 신호를 보조 후보로 검토할 수 있다. 다만 현재 산출물에는 true Boltzmann Energy에 필요한 full logits가 없으므로, 지어냄형 또는 Zero-SE 사례에서 Energy 성능을 실험적으로 주장하지 않는다. 지어냄형이라는 표현은 문헌상 확립된 단일 taxonomy를 대체하려는 용어가 아니라, ``일관성은 높지만 근거성 판단이 남는'' 사례를 설명하기 위한 분석적 구분이다.


\section{SE와 Energy의 상보성}

\subsection{정보 이론적 관점}

SE와 Energy는 서로 다른 정보를 포착한다:

\begin{itemize}
    \item \textbf{SE}: \textit{Inter-response} 다양성 --- 응답들이 의미적으로 얼마나 다른가?
    \item \textbf{Energy}: \textit{Intra-token} 확신도 --- 각 토큰 생성 시 모델이 얼마나 확신하는가?
\end{itemize}

두 신호는 같은 환각 사례의 서로 다른 측면을 본다. 모든 응답이 동일해도(SE=0), 각 토큰의 logit 분포는 여전히 달라질 수 있으므로 응답 간 다양성과 토큰 수준 확신도는 상보적으로 활용될 수 있다.

\subsection{상보성의 수학적 표현}

SE의 정의에서, $SE = 0$이 되는 조건은 모든 응답이 단일 클러스터에 속할 때이다:
\begin{equation}
    SE = -\sum_{c \in C} p(c) \log p(c) = 0 \iff |C| = 1
\end{equation}

이 식은 \textbf{언제 SE가 더 이상 응답 간 구분 정보를 제공하지 못하는가}를 보여준다. $|C|=1$이면 모든 샘플이 하나의 의미 클러스터에 모이므로, SE 값은 구조적으로 0이 되고 같은 구간 안에서는 추가적인 순위화 정보가 거의 남지 않는다. 반대로 $|C|\geq 2$이면 응답 간 의미적 갈라짐이 존재하므로 SE가 본래 의도한 불확실성 신호를 제공한다.

기존 초안은 이 조건에서 Energy로 라우팅하는 hard cascade를 제안했지만, 현재 저장소 산출물에서는 true Boltzmann Energy가 unavailable이다. 따라서 이 논문에서 $|C|=1$ 조건은 Energy 성공을 보인 규칙이 아니라, 향후 full logits 재실험에서 어떤 구간을 별도로 검토해야 하는지 알려주는 분석 조건으로만 사용한다.

\section{Internal Logit과 Confabulation의 관계}

앞 절의 구분을 따르면, Energy는 응답 간 다양성이 사라진 구간에서 생성 토큰의 확신도 패턴을 관찰할 수 있는 후보 신호이다. 즉 Energy는 답의 사실성을 직접 검증하는 지표가 아니라, 겉보기에는 같은 답을 반복하는 사례 안에서도 토큰 선택 과정이 얼마나 불안정하거나 낮은 확신을 보이는지 살필 가능성을 제공한다. 현재 논문에서는 이 가능성을 성능 결과로 주장하지 않고, full logits 보존 후 검증해야 할 향후 분석으로 둔다.

\subsection{Logit 크기와 지식의 관계}

LLM의 logit은 다음 토큰에 대한 상대적 선호를 반영하며, Energy는 이러한 토큰 수준 점수를 집계한 값이다. 따라서 같은 의미의 응답이 반복되는 Zero-SE 구간에서도, 토큰 선택 과정의 확신도 차이는 여전히 남을 수 있다.

\begin{itemize}
    \item \textbf{높은 토큰 확신도}: 특정 표현이 비교적 안정적으로 선택되는 경우
    \item \textbf{낮은 토큰 확신도}: 여러 후보 토큰 사이에서 점수가 분산되는 경우
\end{itemize}

\subsection{Confabulation에서의 Logit 패턴}

본 논문에서 지어냄으로 부르는 행동 범주에서는 다음과 같은 패턴을 기대할 수 있다:
\begin{enumerate}
    \item 반복 샘플링 결과는 비슷하지만, 각 토큰 단계에서는 경쟁 후보 간 점수 차이가 크지 않을 수 있음
    \item 그 결과 응답 간 다양성(SE)만으로는 보이지 않던 차이가 토큰 수준 점수(Energy)에서 드러날 수 있음
    \item 따라서 평균 logit이 낮거나 에너지가 높은 사례는 Zero-SE 내부에서 추가 경고 신호로 해석될 수 있음
\end{enumerate}

따라서 Energy는 SE=0인 상황에서도 보조 판별 신호로 기능할 수 있다. 이러한 해석은 관측 가능한 출력 분포와 점수 패턴에 대한 설명이며, 모델 내부 상태에 대한 논의는 한계 장에서 별도로 다룬다.

\section{Corpus-Grounded Selective Fusion Detector}

현재 실험의 제안 방법은 \textit{Corpus-Grounded Selective Fusion Detector}이다. 이 방법은 SE와 cluster count를 기본 불확실성 특징으로 사용하고, corpus-grounded entity frequency, entity-pair co-occurrence, low-frequency flag, coverage-related proxy를 learned fusion 특징에 추가한다. corpus-risk-only는 단일 corpus signal 기준선이며, 제안 방법 자체가 아니다.

\section{Energy-dependent baseline의 현재 상태}

Energy-only, fixed linear SE/Energy blend, hard cascade는 모두 true Boltzmann Energy가 있어야 평가할 수 있다. 현재 artifact에는 row-level full logits가 없으므로 해당 baseline은 \texttt{full\_logits\_required}와 \texttt{rerun\_required} 상태로 남긴다. 기존 초안에서 사용하던 TruthfulQA-only Energy 숫자와 hard cascade 숫자는 현재 evidence summary의 최종 결과가 아니므로 성능 주장으로 사용하지 않는다.

현재 aggregate AUROC는 SE-only 0.675244, corpus-risk-only 0.499133, learned fusion without corpus 0.477294, learned fusion with corpus 0.444207이다. 따라서 현재 관측 결과는 corpus feature를 추가한 learned fusion이 성능을 높였다는 주장을 지지하지 않는다. 특히 learned fusion with corpus는 learned fusion without corpus보다 AUROC가 0.033087 낮고, robustness bootstrap에서 AUROC 차이의 95\% 구간도 0보다 낮다.

AUPRC 차이는 observed delta가 음수이지만 bootstrap interval이 0을 지나므로 방향성 주장을 하지 않는다. 이 결과는 corpus-grounded signal이 유효하지 않다는 일반 결론이 아니라, 현재 feature export와 leave-one-dataset-out evaluation에서는 corpus fusion headline을 과장할 수 없다는 제한된 결론이다.

\section{Energy와 unavailable baseline}

현재 Energy-only, fixed linear 조합, hard cascade는 모두 \texttt{full\_logits\_required}로 기록된다. true Boltzmann Energy는 softmax 전 full logits 또는 동등한 logsumexp 정보를 행 단위로 보존해야 계산할 수 있다. 현재 산출물의 \verb|semantic_energy_proxy|는 진단용 metadata로 남아 있을 뿐이며, thesis-valid evidence가 아니다.

따라서 본 논문은 기존 초안의 TruthfulQA-only Energy 개선 숫자를 최종 결과로 유지하지 않는다. 향후 재실험은 \verb|uv run python experiments/scripts/run_generation.py| 계열의 repo-owned generation 경로에서 full logits를 보존하고, 그 산출물로 Energy feature와 fusion baseline을 다시 생성해야 한다.

\section{문헌 근거와 guardrail}

Ma Semantic Energy는 low-diversity 또는 single-cluster 상황에서 logit 기반 신호를 검토할 동기를 제공하지만\cite{ma2025}, 현재 저장소 결과의 Energy 성능을 대신 증명하지 않는다. QuCo-RAG는 corpus frequency와 co-occurrence feature의 외부 근거 proxy를 설명하는 참고 문헌이며\cite{qucorag}, corpus count가 모델 내부 지식이나 hidden-state confidence와 같다는 뜻은 아니다. Phillips et al.은 selective prediction과 learned combination의 framing으로만 사용하며\cite{phillips2026}, PC Probe는 구현하거나 재현하지 않았다.

이 구분은 evidence note \path{semantic_energy_single_cluster.md}와 \path{quco_vs_probe.md}의 guardrail을 따른다. 논문 링크 검증은 \verb|uv run|으로 실행하는 thesis evidence validator에서 수행한다.

\chapter{결론}{\label{ch:conclusion}}

\section{연구 요약}

이 논문은 LLM 환각 탐지에서 SE, Semantic Energy 동기, corpus-grounded signal을 결합하려는 실험을 현재 evidence 기준으로 재정리하였다. 핵심 결론은 다음과 같다.

\begin{enumerate}
    \item 현재 aggregate AUROC는 SE-only가 가장 높고, learned fusion with corpus는 learned fusion without corpus보다 낮다.
    \item learned fusion with corpus의 AUROC bootstrap interval은 learned fusion without corpus 대비 0보다 낮아, 현재 산출물은 corpus-fusion 개선 주장을 지지하지 않는다.
    \item AUPRC 차이는 observed only로 다룬다. interval이 0을 지나므로 방향성 결론을 내리지 않는다.
    \item true Boltzmann Energy와 Energy-dependent baselines는 row-level full logits가 없어서 unavailable이다.
    \item 향후 thesis-valid Energy 평가는 uv-managed repo-owned generation과 full logits preservation을 필수 조건으로 삼아야 한다.
\end{enumerate}

\section{학술적 기여}

\begin{enumerate}
    \item \textbf{증거 범위 분리}: 문헌 동기, proxy metadata, 현재 관측 성능을 분리하여 unsupported claim을 제거하였다.
    \item \textbf{정직한 부정 결과 보고}: corpus-grounded learned fusion이 현재 AUROC에서 underperforms한다는 불리한 결과를 본문에 반영하였다.
    \item \textbf{재현성 요구사항 명시}: Energy 평가가 full logits 보존 없이는 성립하지 않음을 실험 방법과 결론에 고정하였다.
    \item \textbf{evidence export 추가}: thesis prose가 특정 JSON summary와 LaTeX table export를 통해 검증 가능하도록 정리하였다.
\end{enumerate}

\section{한계}

본 연구는 다음 한계를 가진다.

\begin{enumerate}
    \item \textbf{Energy 미평가}: 현재 artifact에는 full logits가 없으므로 true Boltzmann Energy와 Energy-dependent baseline 성능을 평가하지 못했다.
    \item \textbf{proxy corpus 의존}: 일부 corpus feature는 캐시 또는 proxy index에서 온다. 따라서 corpus-grounded signal은 외부 근거 proxy일 뿐이며 모델 내부 지식을 직접 증명하지 않는다.
    \item \textbf{회색지대 부재}: 현재 feature table에는 \texttt{AMBIGUOUS\_INCORRECT} 행이 0개이므로, gray-zone threshold sensitivity를 사례 기반으로 주장할 수 없다.
    \item \textbf{모델 내부 상태 해석 불가}: SE, corpus feature, future Energy는 관측 지표이며, 모델의 의도나 내부 원인을 직접 추론하지 않는다.
\end{enumerate}

\section{향후 연구}

\begin{enumerate}
    \item \textbf{repo-owned full-logit generation}: live model run에서 row-level full logits, logsumexp, token ids, provenance를 보존한다.
    \item \textbf{Energy baseline 재생성}: full logits를 확보한 뒤 Energy-only, fixed linear blend, hard cascade를 다시 평가한다.
    \item \textbf{corpus feature 재검토}: corpus feature가 어떤 데이터셋과 label type에서 도움이 되는지 coefficient와 per-dataset error를 함께 분석한다.
    \item \textbf{선택 예측 평가}: Phillips-inspired risk framing은 reference-only로 유지하되, probe 없이 사용할 수 있는 abstention metric을 별도 검증한다.
\end{enumerate}
