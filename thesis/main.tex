\keyword{대규모 언어모델, 환각 탐지, Semantic Entropy, Semantic Energy, Zero-SE 문제, SE-gated Cascade}
\begin{abstract}
대규모 언어모델(LLM)의 환각(hallucination) 문제는 실용적 적용에 있어 가장 심각한 장애물 중 하나이다. Farquhar 등이 Nature(2024)에서 제안한 Semantic Entropy(SE)는 LLM의 의미적 불확실성을 측정하는 대표적 방법이나, 모델이 일관되게 틀린 답변을 생성하는 Zero-SE 문제에서 한계를 보인다.

본 연구에서는 SE와 Semantic Energy의 상보적 특성을 분석하고, SE가 구조적으로 퇴화하는 구간에서 Energy를 보조 신호로 사용하는 SE-gated cascade 방법을 제안한다. 반복 샘플링에서 나타나는 환각 관련 응답 양상은 혼란(confusion)과 지어냄(confabulation)으로 나누어 설명할 수 있으며, SE와 Energy는 서로 다른 구간에서 일부 판별 정보를 제공한다.

TruthfulQA 데이터셋 200개 샘플(K=5)에서의 실험 결과:
\begin{itemize}
    \item Zero-SE 영역이 전체의 19\%를 차지하며, 이 중 73.7\%가 환각
    \item Zero-SE 영역에서 Energy AUROC 0.736 달성 (SE는 판별 불가)
    \item 클러스터 기반 SE-gated cascade는 SE 단독 대비 AUROC +0.029 개선 (0.613에서 0.643)
    \item 합집합 탐지율은 34.1\%이며, 65.9\%의 환각은 두 점수 모두 미탐지
\end{itemize}

본 연구는 SE와 Energy가 서로 다른 환각 관련 응답 패턴을 일부 포착함을 보이고, 두 지표를 제한적으로 결합하는 방법과 그 한계를 함께 제시하여 LLM 환각 탐지 연구에 기여한다.
\end{abstract}

\chapter{서론}

\section{연구 배경}

대규모 언어모델(Large Language Model, LLM)은 텍스트 생성, 질의응답, 요약, 번역 등 다양한 자연어 처리 태스크에서 널리 사용되고 있다. 그러나 최신 LLM도 사실과 다른 내용을 그럴듯하게 생성하는 환각(hallucination) 문제를 보이며, 이는 LLM 신뢰성과 안전성의 핵심 과제로 논의되어 왔다\cite{huang2025survey}. 생성 텍스트의 신뢰성은 사실성(factuality)과 충실성(faithfulness) 관점에서 구분해 다룰 수 있으며\cite{maynez2020faithfulness}, 본 연구는 외부 사실과 충돌하거나 검증 가능한 정답과 맞지 않는 응답을 탐지하는 문제에 초점을 둔다.

환각 문제는 LLM의 실용적 적용에서 중요한 장애물이다. 의료 분야에서 잘못된 진단 정보를 제공하거나, 법률 분야에서 존재하지 않는 판례를 인용하거나, 금융 분야에서 부정확한 수치를 제시하는 경우 큰 피해를 초래할 수 있다. TruthfulQA는 모델이 인간이 흔히 믿는 거짓 정보를 얼마나 그럴듯하게 모방하는지 평가하기 위해 제안된 벤치마크이며\cite{truthfulqa}, 이러한 설정은 LLM 응답을 사후에 검증하고 환각을 자동으로 탐지하는 방법의 필요성을 보여준다.

환각 탐지를 위한 대표적인 불확실성 기반 방법으로 Semantic Entropy(SE)가 있다. SE는 하나의 질문에 대해 여러 응답을 샘플링한 뒤, 자연어 추론(Natural Language Inference, NLI) 모델을 사용하여 의미적으로 클러스터링하고, 클러스터 분포의 엔트로피를 계산한다\cite{farquhar2024}. 반복 샘플링과 응답 간 일관성은 SelfCheckGPT와 같은 선행 연구에서도 환각 탐지를 위한 중요한 단서로 사용되었다\cite{selfcheckgpt2023}. SE가 높으면 모델 응답이 여러 의미로 갈라지므로 의미적 불확실성이 크다는 신호가 된다.

그러나 SE는 응답 간 의미적 다양성에 의존하므로, 모델이 모든 응답에서 같은 의미의 틀린 답변을 반복하면 판별력이 약해진다. 예를 들어 같은 질문을 다섯 번 샘플링했는데 다섯 응답이 모두 동일한 잘못된 인물명이나 수치를 답한다면, 응답들은 의미적으로 완전히 일관되어 하나의 클러스터에 모인다. 이때 SE는 0에 가까워지지만, 낮은 SE가 곧 정답을 의미하지는 않는다. 단지 ``서로 다른 답을 내지 않았다''는 사실만 알려줄 뿐이다. 본 연구는 이처럼 모든 응답이 하나의 의미 클러스터에 속해 SE가 환각 여부를 구분하기 어려워지는 현상을 \textbf{Zero-SE 문제}라고 정의한다.

Zero-SE 설정에서는 ``응답들이 서로 같은가''라는 질문과 ``그 같은 응답이 사실인가''라는 질문이 서로 다른 문제로 분리된다. 이러한 구분은 cascade가 필요한 근거가 된다. 탐지 과정에서는 먼저 SE가 판별 정보를 제공하는 상황인지 확인해야 한다. 응답들이 여러 의미로 갈라지면 SE를 의미적 불확실성 신호로 사용할 수 있고, 모든 응답이 하나의 의미로 수렴하면 SE 대신 남아 있는 보조 신호를 확인해야 한다. 본 연구는 이러한 문제 설정에서 Semantic Energy를 보조 신호로 사용하는 SE-gated cascade를 제안하고, Zero-SE 영역에서 단일 SE의 한계와 결합 방식이 필요한 이유를 실험적으로 확인한다.


\section{연구 목적}

본 연구의 목적은 다음과 같다:

\begin{enumerate}
    \item \textbf{Zero-SE 문제의 정량화}: Zero-SE 현상이 실제로 얼마나 발생하며, 그중 환각 사례가 어느 정도 포함되는지 분석한다.
    
    \item \textbf{탐지 신호의 차이 분석}: SE와 Semantic Energy가 서로 다른 환각 탐지 신호를 제공하는지 검증한다.
    
    \item \textbf{SE-gated Cascade 제안}: 두 메트릭을 상보적으로 결합하여 Zero-SE 영역을 보완하는 방법을 제안한다.
    
    \item \textbf{문헌 기반 위치 정리}: 환각 배경, 불확실성 추정, 사실성 평가 연구와의 관계를 정리하여 제안 방법의 연구 공백을 명확히 한다.
\end{enumerate}

\section{환각 분류, 사실성, 벤치마크}

환각은 LLM이 입력, 문맥, 또는 외부 사실과 맞지 않는 내용을 생성하는 현상으로 다루어진다. Huang 등은 LLM 환각의 원인, 유형, 탐지 과제를 폭넓게 정리하며, 환각 탐지가 LLM 신뢰성 확보를 위한 핵심 문제임을 보인다\cite{huang2025survey}. Maynez 등은 생성 텍스트 평가에서 사실성(factuality)과 충실성(faithfulness)을 구분해 논의했으며\cite{maynez2020faithfulness}, 이 구분은 환각 탐지 문장을 작성할 때 단순한 ``틀림''보다 어떤 기준에서 맞지 않는지 명확히 해야 함을 보여준다.

벤치마크와 평가 연구는 환각 탐지의 대상과 평가 기준을 제공한다. TruthfulQA는 모델이 인간의 오해나 거짓 믿음을 모방하는지를 측정하는 질의응답 벤치마크이다\cite{truthfulqa}. FActScore는 장문 생성에서 원자적 사실 단위의 정밀도를 평가하는 방법을 제안했으며\cite{factscore2023}, AlignScore는 생성 내용과 근거 사이의 정렬을 통해 사실 일관성을 평가한다\cite{alignscore2023}. 이러한 연구들은 환각을 평가하고 분류하는 기준을 제공하지만, 본 연구의 중심인 Zero-SE 설정에서 SE와 Energy를 어떻게 결합할지는 직접 다루지 않는다.

SelfCheckGPT는 외부 지식 베이스 없이 여러 응답을 샘플링하고 응답 간 일관성을 비교해 환각을 탐지하는 접근을 제안했다\cite{selfcheckgpt2023}. 이 흐름은 하나의 응답만 보는 대신, 같은 질문에 대한 여러 생성 결과의 분산과 일관성을 환각 탐지 신호로 본다. Semantic Entropy도 이 관점과 연결되지만, 응답을 의미 클러스터로 묶고 클러스터 분포의 엔트로피를 사용한다는 점에서 더 명시적인 의미 단위 불확실성 측정 방법이다\cite{farquhar2024}.

\section{Semantic Entropy (SE)}

Semantic Entropy는 Farquhar 등이 2024년 Nature에 발표한 LLM 불확실성 측정 방법이다\cite{farquhar2024}. 기존의 토큰 단위 확률 기반 불확실성 측정 방법과 달리, SE는 응답의 의미적 내용을 기반으로 불확실성을 측정한다.

\subsection{작동 원리}

SE의 계산 과정은 다음과 같다:

\begin{enumerate}
    \item \textbf{응답 샘플링}: 하나의 질문 $q$에 대해 LLM으로부터 $K$개의 응답 $\{r_1, r_2, ..., r_K\}$를 샘플링한다.
    
    \item \textbf{의미적 클러스터링}: NLI 모델을 사용하여 응답들을 의미적으로 클러스터링한다. 두 응답이 서로 entailment 관계에 있으면 같은 클러스터로 분류한다.
    
    \item \textbf{엔트로피 계산}: 클러스터 분포의 Shannon entropy를 계산한다.
\end{enumerate}

수식으로 표현하면:
\begin{equation}
    SE = -\sum_{c \in C} p(c) \log p(c)
\end{equation}
여기서 $C$는 의미 클러스터 집합이고, $p(c)$는 클러스터 $c$에 속하는 응답의 비율이다.

\subsection{한계점}

SE의 주요 한계점은 다음과 같다:
\begin{itemize}
    \item \textbf{Hard Clustering}: 응답들을 이진적으로(동일/다름) 분류하여 세밀한 유사도를 반영하지 못한다.
    \item \textbf{Zero-SE 문제}: 모든 응답이 하나의 클러스터에 속하면 SE=0이 되어 판별력이 사라진다.
    \item \textbf{응답 길이와 클러스터링 의존성}: 긴 응답이나 복잡한 답변에서는 NLI 기반 클러스터링 품질에 따라 결과가 달라질 수 있다.
\end{itemize}


\section{SE 계열 확장 방법: KLE와 SNNE}

KLE와 SNNE는 SE의 한계를 완화하려는 후속 방법들이다. Nikitin 등(2024)의 KLE는 hard clustering 대신 의미 커널과 von Neumann Entropy를 사용해 SE를 더 일반적인 불확실성 프레임워크로 확장한다\cite{nikitin2024}. Nguyen 등(2025)의 SNNE는 명시적 클러스터링 없이 응답 간 pairwise semantic similarity를 사용해 긴 응답에서 클러스터 수가 불필요하게 증가하는 문제를 줄이려 한다\cite{nguyen2025}. 두 방법은 SE 계열 연구의 확장 방향을 보여주지만, 본 연구의 초점인 Zero-SE 구간에서 SE와 Energy를 어떻게 선택적으로 결합할지는 직접 다루지 않는다.


\section{Semantic Energy}

Ma 등(2025)이 제안한 Semantic Energy는 토큰 단위의 확신도를 측정하는 방법이다. 현재 문헌 상태에서는 기존 방법 맥락으로만 사용하며, 본 연구의 핵심 주장은 실험에서 관찰한 SE와 Energy의 보완적 신호 차이에 둔다.

\subsection{작동 원리}

Energy는 LLM이 각 토큰을 생성할 때 부여한 \textbf{softmax 전 raw logit} 값을 사용한다:
\begin{equation}
    Energy = \frac{1}{nT} \sum_{i=1}^{n} \sum_{t=1}^{T_i} -z_\theta(x_t^{(i)})
\end{equation}
여기서 $z_\theta(x_t)$는 토큰 $x_t$의 logit 값이다.

\subsection{Zero-SE 보완}

Energy의 중요한 역할은 Zero-SE 영역에서 SE가 잃어버리는 토큰 수준 점수 정보를 제공한다는 점이다. 모든 응답이 동일한 의미라도(SE=0), 각 토큰 생성 시 logit 크기가 다르면 Energy는 추가 판별 신호를 줄 수 있다. 다만 Energy만으로는 응답 간 의미적 다양성 정보를 직접 반영하지 못하므로, SE를 대체하기보다 특정 구간에서 보조하는 방식이 더 적절하다.


\section{기존 연구의 한계 및 연구 공백}

기존 연구들의 한계를 정리하면 다음과 같다:

\begin{enumerate}
    \item \textbf{환각 분류 및 평가 연구}: 환각의 정의, 사실성, 벤치마크를 제공하지만 Zero-SE 영역의 탐지 신호 결합 문제를 직접 다루지 않는다.
    \item \textbf{SE 계열 방법}: 반복 샘플링과 의미 클러스터 분포를 활용하지만, 모든 응답이 하나의 의미 클러스터에 모이는 경우 판별 정보가 부족해진다.
    \item \textbf{Energy 계열 점수}: 토큰 수준 확신도 정보를 제공하지만, 응답 간 의미적 다양성 정보를 직접 활용하지 못한다.
    \item \textbf{결합 구조의 필요성}: 단일 점수로 모든 환각 유형을 처리하기보다, SE가 유효한 구간과 Zero-SE 구간을 나누어 서로 다른 신호를 적용하는 구조가 필요하다.
\end{enumerate}

본 연구는 SE와 Energy의 \textbf{상보성}을 정량화하고, \textbf{SE-gated cascade}를 통해 두 메트릭을 결합하는 방법을 분석한다.

\section{환각이 드러나는 두 가지 양상}

Zero-SE 문제를 이해하려면, 환각이 반복 샘플링에서 서로 다른 방식으로 나타날 수 있음을 구분할 필요가 있다. 하나는 같은 질문에 대해 서로 다른 답변 후보가 나타나는 경우이다. 어떤 샘플에서는 A라고 답하고, 다른 샘플에서는 B라고 답한다면, 모델의 출력은 여러 의미 후보 사이에서 흔들리는 것처럼 보인다. 본 연구에서는 이런 양상을 혼란형으로 부른다. 다른 하나는 반복 샘플링 결과가 일관되지만, 그 답변 자체가 사실이 아니거나 근거 자료로 지지되지 않는 경우이다. 본 연구에서는 이런 양상을 지어냄형 또는 fabrication-like 사례로 구분한다.

이 구분은 문헌의 표준 분류명을 새로 제안하려는 것이 아니라, 기존 연구가 다룬 두 종류의 탐지 단서를 반복 샘플링 설정에 맞게 정리한 것이다. NLG와 LLM 환각 연구는 사실성(factuality), 충실성(faithfulness), 근거와의 일관성, 외부 지식으로 지지되는지 여부를 구분해 다루며\cite{ji2023survey,maynez2020faithfulness,pagnoni2021frank}, 사실성 평가 연구도 생성 문장을 원자적 사실 단위로 나누어 각 사실이 근거 자료에 의해 지지되는지를 평가한다\cite{factscore2023}. 따라서 본 연구에서 말하는 혼란과 지어냄은 모델의 숨은 의도나 내적 상태를 단정하는 표현이 아니라, 여러 번 생성한 출력에서 나타나는 응답 분포와 근거성의 차이를 설명하기 위한 구분이다.

반복 샘플링 기반 연구는 같은 질문에 대해 여러 응답을 생성했을 때 응답 간 불일치나 의미적 분산이 환각 탐지 단서가 될 수 있음을 보였다\cite{selfcheckgpt2023,farquhar2024}. 이는 혼란형 패턴과 잘 맞는다. 응답들이 서로 다른 의미 클러스터로 갈라지면 SE는 그 갈라짐을 엔트로피로 요약할 수 있다. 그러나 이러한 접근은 응답들이 모두 같은 의미로 모이는 경우에는 약해진다. 의미적으로 일관된 출력이 항상 사실적으로 옳다는 뜻은 아니기 때문이다. TruthfulQA는 모델이 그럴듯하지만 거짓인 답변을 낼 수 있음을 평가하고\cite{truthfulqa}, FActScore와 HALoGEN은 생성된 개별 사실이 외부 근거로 지지되는지 또는 fabrication에 해당하는지를 따로 검증한다\cite{factscore2023,ravichander2025halogen}. 즉, \textbf{응답 간 일관성}과 \textbf{정답 또는 근거에 대한 정확성}은 분리될 수 있다.

\subsection{혼란형 환각 (Confusion-like Pattern)}

\textbf{정의}: 같은 질문에 대해 서로 다른 의미의 응답 후보들이 경쟁하는 것으로 관측되는 경우이다. 예를 들어 반복 샘플링 결과가 서로 다른 인물명, 날짜, 원인을 답한다면, 모델 출력은 하나의 안정된 의미로 수렴하지 않는다. 이는 SelfCheckGPT가 활용한 샘플 간 불일치/일관성 비교와, Farquhar 등이 semantic entropy로 포착한 의미적 불확실성 신호와 맞닿아 있다\cite{selfcheckgpt2023,farquhar2024}.

\textbf{특성}:
\begin{itemize}
    \item 반복 샘플링 시 둘 이상의 의미 클러스터가 형성됨
    \item 응답 간 의미적 분산이 커서 정답/오답 후보가 함께 나타날 수 있음
    \item 여러 번 질문 시 \textbf{다양한 응답 패턴}이 관측됨 (상대적으로 높은 SE)
\end{itemize}

탐지 관점에서 SE는 응답 간 다양성을 요약하므로, 의미적 갈라짐이 존재하는 구간에서 우선적인 신호가 된다. 이 범주에서 SE를 사용하는 이유는 ``환각의 원인''을 아는 데 있지 않고, 샘플 간 의미적 불안정성이 관측된다는 점을 점수화하는 데 있다.

\subsection{지어냄형 환각 (Confabulation/Fabrication-like Pattern)}

\textbf{정의}: 반복 샘플링에서 응답이 거의 하나의 의미 패턴으로 수렴하지만, 그 패턴의 사실성 또는 근거 지지 여부는 별도 신호 없이는 판별하기 어려운 경우이다. 예를 들어 모델이 다섯 번 모두 같은 판례명, 논문 제목, 수치를 말하더라도 그것이 실제로 존재하지 않거나 근거 문서에 없는 내용이면 환각일 수 있다. 이때 핵심은 ``모델이 지어내려 했다''는 의도가 아니라, 일관된 출력이 외부 사실이나 근거 자료로 지지되지 않을 수 있다는 점이다. TruthfulQA는 그럴듯하지만 거짓인 답변을 평가 대상으로 삼고\cite{truthfulqa}, FActScore는 생성된 원자적 사실이 신뢰 가능한 근거에 의해 지지되는지를 평가하며\cite{factscore2023}, HALoGEN은 fabrication을 포함한 LLM 환각 오류 유형을 근거 자료와 대조해 분석한다\cite{ravichander2025halogen}.

\textbf{특성}:
\begin{itemize}
    \item 반복 샘플링 시 응답이 단일 의미 클러스터에 가깝게 모임
    \item 응답들이 서로 비슷하므로 응답 간 다양성만으로는 판별 정보가 줄어듦
    \item 여러 번 질문 시 \textbf{일관된 답변}이 반복될 수 있으나, 그 답변이 맞는지 틀린지는 사실성 평가나 토큰 수준 확신도 등 다른 신호가 필요함
\end{itemize}

탐지 관점에서 이 범주에서는 SE가 작아지거나 0이 되어 구분력이 약해질 수 있으므로, Energy와 같은 토큰/로그릿 기반 신호를 보조적으로 사용할 필요가 있다. 지어냄형이라는 표현은 문헌상 확립된 단일 taxonomy를 대체하려는 용어가 아니라, ``일관성은 높지만 근거성 판단이 남는'' Zero-SE 사례를 설명하기 위한 분석적 구분이다. 따라서 혼란형 구간에서는 SE를 사용하고, 지어냄형에 가까운 Zero-SE 구간에서는 Energy와 같은 보조 신호를 함께 고려하도록 cascade를 설계한다.


\section{SE와 Energy의 상보성}

\subsection{정보 이론적 관점}

SE와 Energy는 서로 다른 정보를 포착한다:

\begin{itemize}
    \item \textbf{SE}: \textit{Inter-response} 다양성 --- 응답들이 의미적으로 얼마나 다른가?
    \item \textbf{Energy}: \textit{Intra-token} 확신도 --- 각 토큰 생성 시 모델이 얼마나 확신하는가?
\end{itemize}

두 신호는 같은 환각 사례의 서로 다른 측면을 본다. 모든 응답이 동일해도(SE=0), 각 토큰의 logit 분포는 여전히 달라질 수 있으므로 응답 간 다양성과 토큰 수준 확신도는 상보적으로 활용될 수 있다.

\subsection{상보성의 수학적 표현}

SE의 정의에서, $SE = 0$이 되는 조건은 모든 응답이 단일 클러스터에 속할 때이다:
\begin{equation}
    SE = -\sum_{c \in C} p(c) \log p(c) = 0 \iff |C| = 1
\end{equation}

이 식은 단순한 분류 규칙을 넘어, \textbf{언제 SE가 더 이상 응답 간 구분 정보를 제공하지 못하는가}를 직접 보여준다. $|C|=1$이면 모든 샘플이 하나의 의미 클러스터에 모이므로, SE 값은 구조적으로 0이 되고 같은 구간 안에서는 추가적인 순위화 정보가 거의 남지 않는다. 반대로 $|C|\geq 2$이면 응답 간 의미적 갈라짐이 존재하므로 SE가 본래 의도한 불확실성 신호를 제공한다.

따라서 본 연구의 cascade는 SE가 구조적으로 퇴화하는 경우에 Energy로 판단을 넘기는 제한적 결합 방식으로 해석할 수 있다. 이를 환각 탐지 함수 $D$로 쓰면 다음과 같다:
\begin{equation}
    D(q) = \begin{cases}
        Energy(q) & \text{if } |C| = 1 \text{ (Zero-SE)} \\
        SE(q) & \text{if } |C| \geq 2
    \end{cases}
\end{equation}

이 cascade 함수는 클러스터가 하나뿐인 경우(Zero-SE)에는 Energy를 사용하고, 그 외에는 SE를 유지한다. 즉, 임의의 연속 임계값을 넓게 탐색하기보다 $SE=0 \iff |C|=1$이라는 조건을 구조적 게이트로 사용한다.


\section{Internal Logit과 Confabulation의 관계}

앞 절의 구분을 따르면, Energy는 응답 간 다양성이 사라진 구간에서 생성 토큰의 확신도 패턴을 관찰하는 보조 신호로 사용된다. 즉 Energy는 답의 사실성을 직접 검증하는 지표가 아니라, 겉보기에는 같은 답을 반복하는 Zero-SE 사례 안에서도 토큰 선택 과정이 얼마나 불안정하거나 낮은 확신을 보이는지 살피는 지표이다.

\subsection{Logit 크기와 지식의 관계}

LLM의 logit은 다음 토큰에 대한 상대적 선호를 반영하며, Energy는 이러한 토큰 수준 점수를 집계한 값이다. 따라서 같은 의미의 응답이 반복되는 Zero-SE 구간에서도, 토큰 선택 과정의 확신도 차이는 여전히 남을 수 있다.

\begin{itemize}
    \item \textbf{높은 토큰 확신도}: 특정 표현이 비교적 안정적으로 선택되는 경우
    \item \textbf{낮은 토큰 확신도}: 여러 후보 토큰 사이에서 점수가 분산되는 경우
\end{itemize}

\subsection{Confabulation에서의 Logit 패턴}

본 논문에서 지어냄으로 부르는 행동 범주에서는 다음과 같은 패턴을 기대할 수 있다:
\begin{enumerate}
    \item 반복 샘플링 결과는 비슷하지만, 각 토큰 단계에서는 경쟁 후보 간 점수 차이가 크지 않을 수 있음
    \item 그 결과 응답 간 다양성(SE)만으로는 보이지 않던 차이가 토큰 수준 점수(Energy)에서 드러날 수 있음
    \item 따라서 평균 logit이 낮거나 에너지가 높은 사례는 Zero-SE 내부에서 추가 경고 신호로 해석될 수 있음
\end{enumerate}

따라서 Energy는 SE=0인 상황에서도 보조 판별 신호로 기능할 수 있다. 이러한 해석은 관측 가능한 출력 분포와 점수 패턴에 대한 설명이며, 모델 내부 상태에 대한 논의는 한계 장에서 별도로 다룬다.

\begin{algorithm}[htbp]
\caption{SE-Gated Cascade Detection}

\begin{algorithmic}[1]
\REQUIRE 질문 $q$
\STATE LLM으로부터 $K=5$ 응답 생성
\STATE NLI 클러스터링 수행 후 클러스터 집합 $C$ 산출
\STATE SE 계산: $SE = -\sum_{c} p(c) \log p(c)$
\STATE 토큰 logit 값으로 Energy 계산
\IF{$|C| = 1$}
    \STATE \textbf{return} Energy \COMMENT{Zero-SE: SE가 구분력을 잃는 구간}
\ELSE
    \STATE \textbf{return} SE \COMMENT{의미적 다양성이 존재하는 구간}
\ENDIF
\end{algorithmic}
\end{algorithm}

\subsection{클러스터 기반 판별의 정당성}

제안 방법은 임의의 임계값 $\tau$를 일반적으로 최적화하는 대신, Zero-SE 조건을 직접 나타내는 클러스터 수 $|C|$를 구조적 게이트로 사용한다. 이 선택의 정당성은 다음과 같다:

\begin{enumerate}
    \item \textbf{수학적 직접성}: $SE = 0 \iff |C| = 1$이므로, $|C|=1$은 SE가 구조적으로 퇴화하는 Zero-SE 조건을 바로 가리킨다.
    \item \textbf{제한적 적용 범위}: Energy는 모든 사례에 일괄 적용되지 않고, SE가 응답 간 다양성을 제공하지 못하는 구간에서만 호출된다.
    \item \textbf{해석 가능성}: ``모든 응답이 의미적으로 동일할 때만 보조 신호를 사용한다''는 규칙이므로 Algorithm~\ref{alg:cascade}의 분기 근거를 명확히 설명할 수 있다.
\end{enumerate}

따라서 이 게이트는 SE가 응답 간 정보를 제공하는지를 먼저 판별하는 절차이다. $|C|\geq 2$이면 응답 간 의미적 차이가 있으므로 SE를 그대로 사용하고, $|C|=1$이면 모든 응답이 같은 의미로 모여 SE가 수학적으로 0이 되므로 Energy를 사용한다. 즉 cascade는 Zero-SE 문제에서 바로 따라오는 대응이며, SE를 대체하기보다 SE가 구분력을 잃는 구간을 보완한다.

\section{확장 가능성}

본 연구에서 제안한 클러스터 기반 cascade는 단순하고 해석 가능한 방법이다. 향후 연구에서는 클러스터 수뿐 아니라 SE 값의 연속적 변화를 활용한 가중치 결합 방법을 탐색할 수 있다. 예를 들어, $|C| = 2$인 경계 영역에서 SE와 Energy를 적절히 혼합하는 방식이 가능하다. 다만 이러한 확장은 추가적인 하이퍼파라미터를 도입하므로, 본 연구에서는 클러스터 기반 이진 판별을 기본 형태로 채택하였다.

\chapter{결론}{\label{ch:conclusion}}

\section{연구 요약}

이 논문은 LLM 환각 탐지에서 Semantic Entropy(SE)의 한계로 나타나는 Zero-SE 문제를 정의하고, Semantic Energy를 보조 신호로 결합하는 방법을 제안하였다.

TruthfulQA 데이터셋 200개 샘플에서의 실험을 통해 다음을 확인하였다:

\begin{enumerate}
    \item \textbf{Zero-SE 문제의 심각성}: 전체 샘플의 19\%가 Zero-SE에 해당하며, 이 중 73.7\%가 환각이다.
    
    \item \textbf{Energy의 효과}: Zero-SE 영역에서 Energy가 AUROC 0.736의 별도 판별 신호를 제공한다.
    
    \item \textbf{Cascade의 개선}: 클러스터 기반 SE-gated cascade ($|C|=1$에서 Energy 사용)는 SE-only 대비 AUROC +0.029의 소폭 개선을 보인다 (0.613에서 0.643).
    
    \item \textbf{상보성}: SE와 Energy의 합집합 탐지율은 34.1\%이며, 동시에 65.9\%의 환각은 두 점수 모두 포착하지 못했다.
\end{enumerate}

따라서 본 연구의 결론은 Energy가 SE를 대체한다는 것이 아니다. 보다 구체적으로는, 반복 응답이 하나의 의미 클러스터로 수렴해 SE가 구분 정보를 잃는 경우에도 logit 기반 Energy가 일부 추가 정보를 제공할 수 있으며, 이를 제한적으로 결합하면 작은 성능 개선이 가능하다는 것이다.


\section{학술적 기여}

\begin{enumerate}
    \item \textbf{Zero-SE 문제 정의}: SE 기반 환각 탐지에서 나타나는 Zero-SE 사례를 정의하고 정량화하였다.
    
    \item \textbf{응답 패턴 정리}: 반복 샘플링에서 보이는 응답 다양성과 logit 기반 점수 패턴을 구분해 해석하였다.
    
    \item \textbf{SE-gated Cascade 제안}: 두 메트릭을 제한적으로 결합하는 단순한 방법을 제안하였다.
    
    \item \textbf{이론과 방법 연결}: 기존 문헌의 반복 샘플링 기반 불확실성 논의와 본 실험의 결과 해석을 연결하였다.
\end{enumerate}

\section{한계}

본 연구는 다음 한계를 가진다.

\begin{enumerate}
    \item \textbf{탐색적 표본 규모}: 실험은 TruthfulQA validation split에서 선택한 200개 샘플에 기반한다. 전체 데이터셋이나 다른 벤치마크에 대한 일반화는 추가 실험이 필요하다.

    \item \textbf{높은 환각 비율}: 표본 내 환각은 164개, 정상 응답은 36개로 불균형이 크다. AUROC를 사용했지만, 표본 구성의 편향 가능성은 남아 있다.

    \item \textbf{부분 일치 레이블링}: 정답 문자열과의 부분 일치를 정상 기준으로 사용했기 때문에, 의미적으로 맞지만 표현이 다른 응답이나 부분적으로만 맞는 응답이 잘못 분류될 수 있다.

    \item \textbf{모델과 데이터셋 범위}: 결과는 Qwen2.5-3B-Instruct, DeBERTa-large-mnli, TruthfulQA 설정에 한정된다. 더 큰 모델, 다른 NLI 모델, 다른 데이터셋에서는 Zero-SE 비율과 cascade 효과가 달라질 수 있다.

    \item \textbf{모델 내부 상태 해석 불가}: Energy와 SE는 출력과 logit에서 계산한 관측 지표이다. 이 값만으로 모델의 의도, 지식 보유, 내부 원인을 직접 추론할 수는 없다.
\end{enumerate}


\section{향후 연구}

\begin{enumerate}
    \item \textbf{대규모 데이터셋 검증}: 본 연구는 200개 샘플에서의 탐색적 분석이므로, 전체 데이터셋(817개)으로 확장했을 때의 성능 변화를 검증할 필요가 있다.
    
    \item \textbf{다양한 데이터셋 검증}: TriviaQA, NaturalQuestions, HaluEval 등 다른 데이터셋에서의 일반화 성능 확인이 필요하다.
    
    \item \textbf{더 큰 모델 검증}: Qwen2.5-3B 외에 7B, 72B 등 더 큰 모델에서의 검증이 필요하다.
    
    \item \textbf{경계 영역 처리}: $|C| = 2$인 경계 영역에서 SE와 Energy를 가중 결합하는 방법을 탐색할 필요가 있다.
    
    \item \textbf{Energy 해석 심화}: Energy 점수와 관측 가능한 환각 패턴의 관계를 더 자세히 분석할 필요가 있다.
\end{enumerate}
