# 한국어 학술 어조 규칙

본 문서는 thesis 본문 재작성 시 모든 subagent 가 따라야 할 \emph{한국어 학술 어조} 규칙이다.
용어 치환은 별도 `TERMINOLOGY.md` 를 따른다.
근거: `evaluation.md` §5 한국어 학술논문 글쓰기 원칙 + 본 논문 검토 과정에서 누적된 사용자 지적.

---

## 1. 문장 구조

### 1.1 한 문장 = 한 핵심
한 문장이 두 줄을 넘어가면 분할한다. 접속사·종속절을 줄여 단문 위주로 쓴다.

| 어색 | 자연 |
|---|---|
| "본 결과는 (a) … 임과 (b) … 임을 시사한다." | "본 결과는 두 가지를 시사한다. 첫째, …. 둘째, …." |
| "X 가 Y 보다 큰 진폭을 보인다는 점은, Z 가 W 보다 직접적이라는 해석으로 설명할 수 있다." | "X 가 Y 보다 큰 진폭을 보인다. 이는 Z 가 W 보다 직접적이라는 해석으로 설명할 수 있다." |

### 1.2 명사형 종속절 (`~함을 / ~음을 / ~다는 점을`) 절제
한 문장에 한 번을 넘기지 않는다. 가능하면 단언 분리.

| 어색 | 자연 |
|---|---|
| "X 가 Y 임을 시사함을 의미한다" | "X 는 Y 를 시사한다." |
| "axis 가 신호를 분해함을 보인다" | "이 신호로 문항을 나누면 AUROC 가 구간별로 달라진다." |

### 1.3 영어식 명사화 (`-level / -wise / -based`) 회피
한국어로 풀어 쓴다.

| 영어식 | 한국어 |
|---|---|
| sample-level 신호 | "답변마다 산출한 신호" / "답변 단위로 측정한 신호" |
| prompt-level 신호 | "문항 단위로 산출한 신호" / "한 문항에 하나의 값을 갖는 신호" |
| token-level | "각 토큰에 대해" / "토큰 단위로" |
| score-based 평가 | "신호 점수만으로 평가" |
| dataset-level | "데이터셋별" / "데이터셋 수준의" |

## 2. 시제 / 어조

### 2.1 단언형 위주 (`~이다 / ~한다`)
실험 결과·관찰·정의는 단언형. 미래형 ("~일 것이다") 와 추정형 ("~일 듯하다") 은 자제.

| 어색 | 자연 |
|---|---|
| "이러한 패턴이 관찰될 것이다" | "이러한 패턴이 관찰된다" |
| "~할 수 있을 것으로 기대된다" | "~할 수 있다" |
| "~라고 할 수 있을 것이다" | "~로 볼 수 있다" |

### 2.2 사실 진술 vs 해석 구분
- 사실 (수치·관찰): 단언형 "~이다 / 관찰되었다 / 산출되었다"
- 해석 (인과·함의): "~으로 해석된다 / ~으로 보인다 / ~을 시사한다"
- 추정 (모집단 확장): "~로 해석할 수 있다 / ~의 가능성이 있다"

### 2.3 능동·수동 선택
- 주체가 \emph{본 연구의 행위} 일 때: 능동 ("본 논문은 ~을 분석한다")
- 주체가 \emph{데이터/결과의 상태} 일 때: 수동 ("AUROC 는 0.808 로 측정되었다")

---

## 3. 인칭

- "우리는 / 필자는 / 나는" 금지 → "본 논문은 / 본 연구는"
- "본 논문에서는 ~을 다룬다" 식 표현 OK
- 단 같은 문단에서 "본 논문" 을 반복 남용하면 어색하므로, 두 번째부터는 주어를 생략하거나 능동 동사로 받음.

## 4. 번역투 동사 / 표현 (금지 → 대체)

| 번역투 | 한국어 자연 |
|---|---|
| ~을 갖는다 (have) | ~이다 / ~을 보인다 / ~을 가진다 (직접 소유 한정) |
| ~에 그치다 / 에 그쳤다 | ~에 한정된다 / ~에 머문다 / ~에 머물렀다 |
| ~에 직결된다 | ~과 직접 연결된다 |
| ~을 깨지지 않는다 | ~이 유지된다 |
| ~을 시사함을 보인다 | ~을 보인다 / ~을 시사한다 (둘 중 하나) |
| ~함을 의미한다 (남발) | ~을 뜻한다 / ~을 의미한다 (한 단락 한 번) |
| ~에 대한 분석을 수행하였다 | ~을 분석하였다 |
| ~에 의한 영향 | ~의 영향 |
| ~에 있어서 (in case of) | ~에서 |
| ~으로부터 (from) | ~에서 |
| ~을 위해 (in order to) | ~기 위해 (행위), ~으로 (수단) |
| 매우 / 굉장히 / 정말 | (삭제) 또는 수치로 대체 |
| 사실상 (de facto) | 거의 / 실질적으로 |
| 한편 (on the other hand) | (앞 문장과 대조 명확할 때만) |
| 결과적으로 (as a result) | "그 결과 / 따라서" |

## 5. 단정 vs 과장

| 과장 / 주관 | 학술적 |
|---|---|
| 최강 분해력 | 가장 큰 변동 폭 |
| 최약 분해력 | 가장 작은 변동 폭 |
| 매우 약하다 / 매우 강하다 | 제한적이다 / 큰 폭이다 |
| 강력한 신호 | 신뢰도 높은 신호 / 분리력이 큰 신호 |
| 현저히 크다 | 큰 폭으로 크다 / 수치로 직접 비교 |
| 흥미로운 점은 / 흥미롭게도 | (삭제) 또는 "주목할 점은" |
| 오히려 더 크게 (역접) | "더 크게" 만 |
| 완전히 (totally) | "거의 완전히" 또는 (삭제) |
| 핵심 통찰 | 핵심 논지 / 핵심 주장 |
| 심각한 오류 | 중대한 오류 |

## 6. 영어 직삽 / lab-노트체 (금지)

### 6.1 영어 복합 표현 직삽 금지
| 금지 | 대체 |
|---|---|
| multi-axis 분해 | "X 신호의 구간별 변동" / "여러 신호의 구간별 비교" |
| paper-faithful 하게 구현 | "원 논문 그대로 구현" / "동일하게 구현" |
| motivation 정당화 | "후속 분석의 동기 제시" |
| trivial-but-quantified | "자명하지만 정량화가 필요한" |
| Robustness check — | "강건성 검증:" |
| Effect-size precision | "효과 크기의 정밀도" |
| reference comparison | "참고용 비교" |
| in-context knowledge | "사전학습된 지식" |
| sampling variability | "표본 변동성" |
| selection bias | "선택 편향" |
| ↔ 화살표 | 텍스트 연결어 ("~과 ~ 사이의 비대칭") |

### 6.2 paragraph / section heading 에 부연 금지
| 금지 | 대체 |
|---|---|
| `\section{... : 본 논문 Main 발견}` | `\section{Multi-Axis 분해 결과}` |
| `\paragraph{발견 1 (entity 쌍 axis 가 최강 분해력).}` | `\paragraph{Entity 쌍 동시 등장 신호의 변동 폭.}` |
| `\paragraph{... (개선됨)}` | (라벨 삭제) |
| `\paragraph{... (Baseline)}` | `\paragraph{Entity 수준 신호.}` |

### 6.3 부연 괄호 금지
본문 끝에 짧은 메모성 괄호 (`(... 차단)`, `(... 보고됨)`, `(개선됨)` 등) 사용 금지.
필요한 보충은 다음 문장으로 분리.

| 금지 | 대체 |
|---|---|
| "같은 fold 에 배치되도록 강제한다 (prompt 단위 leakage 차단)." | "같은 fold 에 배치하여 prompt 단위 정보 누출을 방지한다." |
| "재생성하였다 (AUROC 변화는 ±0.001 로 미미함을 확인)." | "재생성하였다. 재생성 전후 AUROC 변화는 ±0.001 범위에 머물렀다." |

### 6.4 자기 강조 \emph{} 금지
\emph{체계적으로 제안}, \emph{참조 형식}, \emph{단위} 같은 자평적 강조 금지.
\emph{} 는 \emph{학술 용어 정의 시 한 번} 만 사용 (예: "본 논문은 \emph{선택 편향} (selection bias) 이라 부른다").

## 8. 인용 / 수식 / 표 인용

- 표/그림 인용: `표~\ref{...}` / `그림~\ref{...}` (hardcoded 번호 금지)
- 인용: `Farquhar 등 (2024)\cite{...}` (저자 등 + 연도 + cite)
- 수치 매크로 사용: `$\Delta$\HeadlineEntityPairSEDelta{}` 형식 통일
- 통계 인용: `Spearman $\rho$=+0.648 ($p$=0.043)` / `95\% 신뢰구간 [+0.002, +0.117]` 형식
- 신뢰구간: "95\% 신뢰구간" / "95\% CI" 중 \emph{전체 본문 통일} (CI 사용 시 첫 등장에 정의)

## 9. 문단 구조

- 한 문단 = 한 주제, 3--7 문장
- 문단 첫 문장에서 주제 제시, 중간에 근거·설명, 마지막에 함의 (필요 시)
- 결과 단락은 "수치 → 비교 → 해석" 순서, 해석을 결과보다 먼저 두지 않음

## 10. 자주 쓰기 좋은 학술 표현 (`evaluation.md` §5.2 발췌)

| 상황 | 표현 |
|---|---|
| 연구 목적 | "본 연구의 목적은 ~을 분석하는 데 있다." |
| 연구 질문 | "본 연구는 다음의 연구 질문을 설정한다." |
| 선행 연구 | "기존 연구에서는 ~에 주목하였다." |
| 선행 연구 한계 | "그러나 기존 연구는 ~을 충분히 다루지 못하였다." |
| 방법 | "이를 위해 본 연구는 ~ 방법을 사용한다." |
| 결과 제시 | "분석 결과, ~한 경향이 나타났다." |
| 해석 제시 | "이는 ~을 시사한다." |
| 한계 제시 | "본 연구는 다음과 같은 한계를 지닌다." |
| 결론 제시 | "따라서 본 연구는 ~로 결론 내릴 수 있다." |

## 11. 자기 점검 체크리스트 (재작성 후 확인)

- [ ] 한 문장이 세 줄을 넘는가? → 분할
- [ ] "갖는다 / 그친다 / 직결된다" 가 남아 있는가? → 교체
- [ ] "최강 / 최약 / 매우 / 현저히 / 흥미롭게도" 가 남아 있는가? → 삭제 또는 수치화
- [ ] 영어 단어가 일관 없이 섞여 있는가? → `TERMINOLOGY.md` 와 대조
- [ ] paragraph heading 에 괄호 부연이 있는가? → 명사구만 남김
- [ ] 코드 식별자 (`*_axis`, `prompt_id` 등) 가 본문에 있는가? → 한국어로 풀어 씀
- [ ] 한 단락에 "~함을 시사한다 / 의미한다" 가 두 번 이상 나오는가? → 한 번으로 줄임
- [ ] "본 논문 / 본 연구" 가 같은 문단에 세 번 이상인가? → 두 번째부터 주어 생략
